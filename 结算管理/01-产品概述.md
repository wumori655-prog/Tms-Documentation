# 第一章 产品概述

## 1.1 产品定位

结算列表是结算管理模块的主入口页面，服务于赛事结算的全流程管理。

| **定位项** | **说明**                                |
| ------- | ------------------------------------- |
| 所属系统    | P1 体育操盘系统                             |
| 核心目标    | 展示待结算赛事全貌，支持盘口级人工结算，分配结算人员            |
| 上游      | 操盘管理（赛事/盘口/投注数据）、数据源管理(IM 比分/盘口状态推送)  |
| 下游      | 结算详情页（单场赛事结算操作）、财务系统（派彩发放）、风控管理（异常告警） |
| 数据源口径   | 仅 IM（Inplay Matrix）为唯一数据源             |
| 结算优先级   | 结算详情页优先。若注单视图和详情页同时操作同一盘口，以详情页为准      |


## 1.1a 本期范围声明（一期实装边界）

> **本期实装**：
>
> - **数据源自动拉取比分数据**（基础数据能力）—— 本地系统按 IM 接口拉取半场 / 全场 / 进球 / 角球 / 红黄牌等比分字段，供结算详情页赛果录入弹窗预填与核对
> - **3 弹窗赛果录入**（半场 / 15 分钟段 / 全场，字段归属严格分治）
> - **手动结算路径**（L3 录入或核对比分 → 系统按规则预填盘口结果 → 结算人员逐盘口手动确认 / L4 逐盘口手动选结果）
> - **结算人员分配**（本列表完成）、**结算列表查看**、**赛事驳回入口**（入口在结算详情页）
>
> **本期不实装**（文档保留作为后续迭代设计参考）：
>
> | 方向 | 本期范围 | 文档引用 |
> |-----|---------|---------|
> | 基于比分的自动结算业务动作（L1 数据源盘口自动派彩 / L2 可操盘盘口自动应用赛果 + 自动派彩 / 串关母单自动重算 + 自动差额处理 / 自动重算聚合窗口） | **不实装**：数据源虽然本期自动拉取比分，但所有"应用赛果"、"派彩"、"差额处理"动作一律由结算人员手动触发 | 结算详情页第 11.0.1 节 |
> | 风控管理独立界面 | **不做**：风控参数（限额 / 告警 / 赔率边界 / RTP / 调幅限制等）由体育后台下发到操盘后台后在结算场景直接消费，本模块不提供风控配置入口 | 结算详情页第 20 章 配置项汇总 |
> | 结算报表模块 | **不做**：一期结算完成的数据仅在本模块与结算详情页内留存并支持审计查询；结算报表作为后续独立迭代模块 | 结算报表 index.md |
>
> **核心区分**：
>
> - "**数据源自动拉取比分**"（基础数据获取）= 本期实装
> - "**自动结算 / 自动派彩 / 自动重算 / 自动差额处理**"（基于比分的自动业务动作）= 本期不实装，一律由结算人员手动触发

## 1.2 目标用户

| **角色**   | **职责**       | **核心操作**                                  |
| -------- | ------------ | ----------------------------------------- |
| **结算人员** | 一线结算操作       | 在注单视图中为盘口选择结算结果、发起盘口二次结算（入口在详情页）、处理赛事异常   |
| **结算主管** | 管理级操作 + 一线操作 | 含结算人员全部操作 + 结算人员分配 / 批量分配                 |
| 风控人员     | 异常赛事审查       | 查看异常赛事、风控告警处理（只读）                         |
| 财务人员     | 结算数据查看       | 查看赛事列表和注单列表（只读）                           |
| 系统       | 自动结算         | IM 比分推送 → 本地按规则自动结算、BT6 比分变动触发结算、串关母单自动结算 |

> **角色对齐**：结算模块的角色体系与操盘列表对应：结算人员 ≈ 操盘手（一线），结算主管 ≈ 主管（管理）。同一自然人可承担两个岗位，按操作页面识别角色（操盘页面用操盘手身份；结算页面用结算人员身份）。

## 1.3 核心价值

结算列表解决三个问题：

**问题一：哪些赛事需要处理？**

赛事开赛后自动出现在结算列表。结算人员打开页面即可看到所有待结算赛事，异常赛事红色高亮，一目了然。

**问题二：哪些注单需要人工结算？**

切换到注单视图，按「无赛果原因」筛选，直接看到所有需要人工介入的注单。L1/L2 由系统自动处理，结算人员只需关注 L3/L4。

**问题三：谁负责哪场赛事的结算？**

赛前和滚球阶段可分配不同结算人员，结算主管可批量分配。

## 1.4a 页面展示范围

本页面（结算管理 · 赛事视图 / 注单视图）仅展示**进行中**的结算业务，不展示已完成的结算记录。

| **展示规则**      | **说明**                  |
| ------------- | ----------------------- |
| 仅展示待结算 + 不可结算 | 这两类是结算人员需要主动处理的对象       |
| ~~已结算不在本页展示~~ | ~~已结算的赛事/注单需透过结算详情页查询~~ |

> **设计原因**：结算管理是结算人员的「**主战场**」，应只显示需要处理的事项。已完成的历史纪录另开查询入口，避免页面过载。

## 1.5 与操盘模块的关系

**赛事双列表并存**

| **阶段**                | **操盘列表** | **结算列表**           | **说明**                        |
| --------------------- | -------- | ------------------ | ----------------------------- |
| 早盘（距开赛 \ 30 分钟）       | 存在       | **不存在**            | 盘口开放投注；结算列表尚无结算任务             |
| **即将开赛（距开赛 ≤ 30 分钟）** | 存在       | **存在**（自动进入，待结算筛选） | **双列表并存起点**：结算列表提前收入，便于结算人员准备 |
| 已开赛（滚球中）              | 存在       | 存在                 | 盘口陆续关盘后结算人员开始逐盘口结算            |
| 比赛结束（完场但未结算）          | 存在       | 存在                 | 结算人员赛果录入 + 逐盘口确认              |
| 赛事级确认结算后              | **移除**   | 存在（归档到结算日期）        | 结算人员点击「结算赛事」触发                |


**分界规则（硬约束）**

| **规则**            | **说明**                                                                           |
| ----------------- | -------------------------------------------------------------------------------- |
| **即将开赛自动进入**（硬约束） | 赛事一旦进入即将开赛阶段（**距开赛 ≤ 30 分钟**，与操盘列表"即将"Tab 阈值一致），**自动**出现在结算列表"待结算"筛选结果           |
| 早盘排除（硬约束）         | 距开赛 \ 30 分钟的早盘赛事**不进入结算列表**，即使开赛时间落在工具栏时间筛选范围内。早盘赛事的盘口投注管理在操盘列表进行                |
| 即将开赛阶段的操作限制       | 结算列表中处于"即将开赛（未开赛）"的赛事仅可**查看**，不能发起任何结算操作（此时盘口未关盘，无结算任务）。实际结算从盘口陆续关盘开始            |
| 盘口关盘 → 盘口可结算      | 盘口关盘后（数据源 IM 推送关盘）该盘口可结算，但实际结算需等赛果可得                                             |
| 赛事级确认结算           | 结算人员点击「结算赛事」后，赛事从**操盘列表移除**，结算列表保留（归档到结算日期）                                      |
| 赛事级驳回             | 赛事重新出现在操盘列表（已结算 / 已取消 / 异常均可驳回 → 待结算；距最新结算时间 ≤ 24h）                              |
| 阈值术语来源            | "即将开赛"阈值 30 分钟对齐操盘列表第 5 章 5.2.2 节即将开赛的定义；"紧急"10 分钟阈值仅用于操盘列表一键锁盘 / 置顶，**不引入结算列表** |


## 1.6 结算自动化 4 层模型

> 完整定义见结算详情页第 1 章。以下为列表视角的简要说明。

| **层级** | **条件**                         | **注单视图中的操作**                                                                                      |
| ------ | ------------------------------ | ------------------------------------------------------------------------------------------------- |
| L1     | 数据源盘口 + IM 推送盘口终态（比分 + 状态字段稳定） | 已结算，显示结果（盘口二次结算在详情页）                                                                              |
| L2     | 可操盘盘口 + IM 推送比分数据              | 「确认结算」（系统按 BT 级判定规则预填，可修改）。**两种子路径**：自动结算开关 = 开 → 本地按规则自动计算并确认结算（无需人工）；= 关 → 预填后人工确认。详见结算详情页 01.5 |
| L3     | 可操盘盘口 + IM 未推送比分（或推送延迟）        | 「人工结算」（详情页录入比分→系统按规则计算；列表快捷选结果）                                                                   |
| L4     | IM 无法覆盖 BT（40种，本地规则无法自动推导）     | 「人工结算」（逐盘口选择赢/输/走水）                                                                               |

> **核心概念澄清**：IM 推送的是**比分/进球/角球/红黄牌**等原始数据 + **盘口状态字段**（开盘/关盘/已结算标记），**不直接推送"赢/输/走水"结算结果**。本地系统基于 IM 推送数据 + BT 级判定规则自动计算结算结果（数据源盘口按 IM 状态字段直接驱动；可操盘盘口按比分+规则算）。

## 1.7 页面顶部提示

页面顶部固定展示紫色提示条：

> 💡 **结算模块说明**：本模块按注单结算自动化层级（L1-L4）分配结算人员介入程度——
>
> - **L1**（数据源盘口 + IM 已推送盘口终态）：本地依 IM 状态直接派彩，**无需人工介入**
> - **L2**（可操盘盘口 + IM 已推送比分，自动结算开关=开）：本地按 BT 级判定规则自动计算并派彩，**无需人工介入**
> - **L2**（可操盘盘口 + IM 已推送比分，自动结算开关=关）：本地按规则预填结果，**需结算人员确认**
> - **L3**（可操盘盘口 + IM 未推送比分）：**结算人员补录比分**，本地按规则计算结果
> - **L4**（IM 无法覆盖 BT，40 种小众玩法）：**结算人员逐盘口手动选择结果**（本地规则无法自动推导）
>
> 「需人工结算」的注单（L2 人工确认 / L3 / L4）优先展示，请及时处理。
