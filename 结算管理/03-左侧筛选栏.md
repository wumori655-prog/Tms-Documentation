# 第三章 左侧筛选栏

## 3.1 筛选栏结构

左侧固定宽度(220px),各筛选组纵向排列。每个选项右侧显示实时计数徽章。组内单选(含「全部」选项)。

> **数据范围硬约束**:本筛选栏展示的所有赛事均为**已进入即将开赛阶段(距开赛 ≤ 30 分钟)或已开赛**的赛事(与操盘列表"赛前"阶段区分)。**赛前赛事(距开赛 > 30 分钟)不进入结算列表任何筛选结果**。若工具栏时间筛选范围包含更早时段,结果仍只显示其中已进入即将开赛或已开赛的赛事。详见第 1 章 1.5 分界规则。
>
> **术语对齐**(全局三段论,核心规则源自操盘列表 9.2 比赛进程派生规则):**赛前**(距开赛 > 30 分钟,仅操盘列表管理)→ **即将**(距开赛 ≤ 30 分钟,双列表并存起点)→ **滚球**(已开赛)。本筛选栏 3.2 节"赛事阶段"筛选值采用"即将 / 滚球"两档,不含"赛前"(赛前赛事不进入本列表)。

## 3.2 筛选条件表

| **筛选组** | **选项** | **默认值** | **计数徽章颜色** | **说明** |
| ------- | ------ | ------- | ---------- | ------ |
| 赛事结算状态 | 全部赛事 / 待结算赛事 / 已结算赛事 / 异常赛事 / 已取消赛事 | 全部赛事 | 全部灰 / 待结算橙 / 已结算绿 / 异常红 / 已取消灰 | 各选项详细定义见 3.2.1~3.2.5 |
| 赛事阶段 | 全部 / 即将 / 滚球(进行中) | 全部 | 灰 / 蓝 / 红 | 各选项详细定义见 3.2.6~3.2.7 |
| 运动类型 | 全部运动 / 足球 / 篮球 | 全部运动 | 灰 | 一期仅足球 |

> **v1.3 修订**:撤销原「已结算」删除线——经业务复核确认,「已结算赛事」作为筛选项继续保留(默认 24h 驳回窗口内可见,详见 3.2.3)。

---

## 3.2.1 全部赛事

| 项 | 内容 |
|---|---|
| **含义** | 筛选栏首选项,显示当前结算列表范围内的所有赛事 |
| **计数口径** | 已进入即将开赛(≤30分钟未开赛)+ 已开赛(滚球中)的所有赛事,涵盖以下所有结算状态:待结算 / 已结算 / 异常 / 已取消 |
| **不计入** | 赛前赛事(距开赛 > 30 分钟)——这些赛事在操盘列表管理,不进入结算列表 |
| **徽章颜色** | 灰色 |
| **默认值** | ✅ 默认选中 |
| **联动** | 与「赛事阶段」「运动类型」AND 关系;选中后下方主列表展示全部赛事 |

---

## 3.2.2 待结算赛事

| 项 | 内容 |
|---|---|
| **含义** | 赛事尚未完成结算确认 |
| **计数口径** | 赛事级状态 = `待结算`(包括"即将开赛"、"滚球中"、"所有盘口已结算但未点击确认结算"的过渡态) |
| **不计入** | 异常态(归到 3.2.4)/ 已取消(归到 3.2.5)/ 已结算(归到 3.2.3) |
| **徽章颜色** | 橙色 |
| **联动** | 待结算赛事是结算工作的主要处理对象,通常配合"赛事阶段"做进一步筛选(如:待结算 + 滚球) |
| **状态来源** | 详见第 12 章 12.1 赛事状态流转 |

---

## 3.2.3 已结算赛事

| 项 | 内容 |
|---|---|
| **含义** | 赛事已经过结算人员手动确认结算 |
| **计数口径** | 赛事级状态 = `已结算`,且在驳回窗口内 |
| **驳回窗口** | 距赛事**最近一次结算时间**(UTC 时间戳)≤ 24 小时内,赛事仍可见,允许执行「赛事驳回」(详见第 12 章 12.0 节及结算详情 15.3) |
| **超窗口处理** | 超过 24h 后,赛事终态永久化,从结算列表归档移除(按结算日期归档) |
| **徽章颜色** | 绿色 |
| **重新结算的影响** | 若执行了赛事驳回 → 该赛事回到"待结算",24h 窗口**向后滚动**(以最新结算时间为准重新计算) |
| **状态枚举来源** | 详见第 11 章 11.1 赛事结算状态(SSOT 源头) |
| **状态流转规则** | 详见第 12 章 12.1 赛事状态流转 |

> **设计意图**:已结算赛事保留 24h 是为了提供"驳回机会"——发现结算错误时可通过【赛事驳回】+【盘口二次结算】组合操作修正。超过 24h 后系统判定为永久确认,不再可改。

---

## 3.2.4 异常赛事

| 项 | 内容 |
|---|---|
| **含义** | 比赛进行中遇到中断,但尚未完成最终处理 |
| **计数口径** | 赛事级状态 = `异常态`,具体异常类型 ∈ {腰斩暂缓, 延期暂缓} |
| **不计入** | 数据错误(数据错误仅做审计标记,不改变赛事状态,详见 12.0 节) |
| **触发后行为** | 进入异常态后,系统会**立即批量关盘**该赛事所有开盘中盘口,但订单暂不动 |
| **后续流转** | 异常赛事经过取消/腰斩全额退款/延期超时退款处理后,转为「已取消」(归到 3.2.5) |
| **徽章颜色** | 红色 |
| **与"已取消"的区别** | 异常赛事 = **中间过程态**(已识别中断,暂缓处理);已取消 = **终态**(已完成取消、退款流程) |
| **联动** | 异常赛事与"赛事阶段"两组独立判断——异常赛事可同时是"即将"或"滚球",AND 关系下两组各自筛选 |
| **状态枚举来源** | 详见第 11 章 11.1 赛事结算状态(异常态枚举值 = exception) |
| **异常处理流程** | 详见第 12 章 12.0 节异常分支 |

---

## 3.2.5 已取消赛事

| 项 | 内容 |
|---|---|
| **含义** | 赛事已通过异常处理流程完成取消,所有盘口已退款 |
| **计数口径** | 赛事级状态 = `已取消` |
| **触发场景** | 赛事取消 / 腰斩全额退款 / 延期超时退款 |
| **盘口状态** | 该赛事所有盘口已退款,注单已通过派彩回收机制处理(详见第 15.4 节) |
| **徽章颜色** | 灰色 |
| **与"异常赛事"的区别** | 已取消 = **终态**;异常赛事 = **过程态**——异常需经过取消流程才转为已取消 |
| **状态枚举来源** | 详见第 11 章 11.1 赛事结算状态(已取消枚举值 = cancelled) |
| **取消流程来源** | 详见第 12 章 12.0 节异常分支 |

---

## 3.2.6 即将(赛事阶段组)

| 项 | 内容 |
|---|---|
| **含义** | 赛事即将开赛,但尚未开赛 |
| **计数口径** | 距开赛时间 ≤ 30 分钟 **且** 尚未开赛 |
| **徽章颜色** | 蓝色 |
| **联动** | 与「赛事结算状态」组 AND 关系——例如"待结算 + 即将"= 当前即将开赛且未完成结算的赛事 |
| **与异常态的关系** | 异常态独立于阶段——一个赛事可以同时是"即将"且"异常",在两组筛选里**各自计入**(不互斥) |
| **术语来源** | 全局三段论,详见 3.1 节 |

---

## 3.2.7 滚球(赛事阶段组)

| 项 | 内容 |
|---|---|
| **含义** | 赛事已开赛,比赛进行中 |
| **计数口径** | 当前服务器时间 ≥ 开赛时间 |
| **包含范围** | 所有比赛进程中的赛事(包括各类盘口陆续关盘/结算的中间状态) |
| **徽章颜色** | 红色 |
| **联动** | 与「赛事结算状态」组 AND 关系 |
| **与异常态的关系** | 同 3.2.6——异常赛事可同时是"滚球",两组筛选独立判断 |
| **术语来源** | 全局三段论,详见 3.1 节 |

---

## 3.3 筛选联动规则

| **规则项** | **说明** |
| ------- | ------ |
| 各组之间 | AND 关系,同时满足 |
| 组内选项 | 单选(含「全部」),互斥 |
| 切换选项 | 点击立即生效,刷新主内容区(赛事列表或注单列表) |
| 计数更新 | 切换某组后,其他组的计数同步更新(反映当前筛选下的分布) |
| 跨视图 | 左侧筛选栏始终显示赛事计数,不随赛事视图/注单视图切换而变化 |

## 3.4 异常赛事展示

异常赛事已独立为筛选项(红色计数),同时在列表中视觉区分:

| **展示位置** | **展示方式** |
| --------- | --------- |
| 左侧筛选栏「异常赛事」 | 红色计数徽章,独立筛选 |
| 赛事视图列表 | 异常赛事卡片红色边框 + 红色状态标签 |

## 3.5 日期筛选默认值

| **筛选项** | **默认值** | **说明** |
| ------- | ------- | ------ |
| 日期范围 | **今天** | 按开赛时间筛选,预设显示当天的赛事 |

> **说明**:用户可手动调整日期范围,时间快捷按钮提供「**今天 / 昨天 / 本周**」三个选项,详见 [第 4 章 4.4 时间快捷按钮](https://github.com/wumori655-prog/Tms-Documentation/blob/main/%E7%BB%93%E7%AE%97%E7%AE%A1%E7%90%86/04-%E5%B7%A5%E5%85%B7%E6%A0%8F%E6%A8%A1%E5%9D%97.md)。
