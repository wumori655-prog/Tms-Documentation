# 第七章 盘口结算卡片

> **关盘口径（2026-04-21 生效）**：关盘来源统一为数据源推送（唯一来源）；关盘 = 绝对终态，操盘页与结算详情页均不提供人工开盘入口。

> 每个盘口在结算详情页中以卡片形式展示。卡片结构统一，内部渲染根据渲染器类型不同而不同。

## 7.2 卡片 Header

| 元素 | 说明 |
|------|------|
| 盘口名称 | 如「全场让球」 |
| BT 编号 | 如「BT1」 |
| 数据源标签（4 态）| **绿色「数据源锁定」**（数据源正常接管，已推送盘口终态数据 + 状态字段，只读）/ **橙色「数据源缺失」**（数据源超时未推送盘口终态，解锁为待手动结算）/ **蓝色「本地操盘」**（UI 简写，即可操盘盘口）/ **紫色「BT273 透传」**（透传类盘口，结算结果由数据源直接下发）|

| 投注额 | 该盘口下注单总投注额 |

| 取消按钮 | 盘口级取消，弹出赛事异常处理弹窗（盘口级仅使用取消原因，不涉及延期/腰斩子方案）|

## 7.3 卡片 Body — 按渲染器类型

### 7.3.1 MultiLineTable（让球/大小类）

按盘口线分行，每行两个选项：

| 列 | 说明 |
|----|------|
| 盘口线 | 如 -0.5、-0.25、-1.0 |
| 选项 A | 主队/大 + 结算结果下拉框 |
| 选项 B | 客队/小 + 结算结果下拉框（自动配对联动） |
| 主力线标记 | 主力线行加粗显示 |
| 操作 | 待结算：「结算」按钮；已结算：灰色「已结算」+ 「二次结算」按钮 |

**配对联动规则**：选择选项 A 的结算结果后，选项 B 自动填入对应结果（赢↔输、赢半↔输半、走水↔走水、取消↔取消）。

**盘口线级独立**：同一玩法下不同盘口线可处于不同结算状态。

### 7.3.2 SingleLineTable（独赢/单双/双重机会等）

所有选项在同一区域内展示：

| 列 | 说明 |
|----|------|
| 选项名 | 如 主胜 / 和局 / 客胜 |
| 结算结果 | 下拉框（赢/输/取消/退款，不含赢半/输半/走水） |
| 操作 | 玩法级一个「结算」按钮 |

### 7.3.3 Matrix（波胆/反波胆/半全场）

**BT6 波胆**：5×5 矩阵 + "其他"选项。每个单元格 = 一个选项（精确比分）。命中的选项标绿，其余标灰。

**BT158 反波胆**：5×5 矩阵 + "其他"选项（共 26 个，与 BT6 取反的结构）。判定规则：选项比分 ≠ 终场比分 → 赢；选项比分 = 终场比分 → 输。命中（赢）的选项标绿，未命中（输，唯一 1 个）的选项标红。无需玩法级特殊录入，依赖赛事级终场比分即可自动判定。

**BT9 半全场**：3×3 矩阵（9 种半场/全场结果组合）。

### 7.3.4 LongList（总进球）

4 列网格展示 0~7+ 球共 8 个选项。命中的选项标绿，其余标灰。

## 7.3.6 自适应基准分组结算规则（核心规则）

> 本节是整个结算模块处理"同盘口多组基准分"场景的核心规则。所有相关章节（附录 N7 / 映射表 / 第 9.4 节 / settlement 第 6.1 节/第 6.8 节/第 7.3 节/第 8.1.1 节 / 第 12.15 节）均引用本节。

### 7.3.6.1 背景

滚球让球/大小类盘口的结算基准分 = 终场比分 − 下注时比分（每笔注单独立）。

**关键业务约束**：
- 盘口不支持本地新建盘口线，所有盘口数据来自 IM 数据源
- 当 IM 在比分变化后对该盘口走"暂停-恢复"（同盘口 ID 继续接单）时，同盘口内会出现多笔"下注时比分"不同的注单
- 此时同盘口内按基准分会自然划分为多组，各组结算结果**可能不同且都正确**

**适用范围（硬约束）**：
- ✅ 本节规则**仅适用于常规单场赛事**（EventTypeId = 2）的滚球让球 / 大小类盘口（BT1 / BT2 / BT32 / BT160 / BT161 / BT179 / BT180，及其半场变体）
- ❌ 本节规则**不适用于冠军赛事 / 长期盘口**（EventTypeId = 1，如夺冠球队、晋级队伍、最佳球员等），冠军盘口按独立的 [冠军赛事结算详情](/settlement-outright/) 模块处理，不涉及"下注时比分"概念
- ❌ 本节规则**不适用于独赢 / 波胆 / 总进球 / 双重机会 / 半全场等不受滚球影响的 BT**（BT3 / BT5 / BT6 / BT7 / BT8 / BT9 / BT158 / BT159），这些 BT 无论赛前或滚球均用终场比分结算，不存在基准分组概念

**单关与串关子单的同构性（SSOT）**：

盘口级结算判定不区分注单来源（单关 vs 串关子单），仅按"盘口线 + 基准分组"划分作用域。具体对应关系：

| 维度 | 单关注单 | 串关子单 |
|------|---------|---------|
| 结算判定粒度 | 盘口线 + 基准分组 | 盘口线 + 基准分组 |
| 多组场景入口 | 注单视图·单关注单行展开后弹窗（按组分区）| 注单视图·串关展开面板（每行代表所属的一个组）|
| 跨组批处理入口 | 弹窗内各组独立结算按钮 | 统一导流到结算详情页盘口卡片（串关展开面板不复制多组分区 UI）|
| 配对联动作用域 | 同盘口线 + 同基准分组 | 同盘口线 + 同基准分组 |
| 差额基线（二次结算）| 组级独立（见 [15.4.4](/settlement-detail/15-驳回与二次结算)）| 组级独立（同） |

**口径解释**：串关子单的结算本质上仍是盘口级判定，只是注单归属的母单是一串多场赛事组合。盘口内一笔子单被选定结果后，该盘口内同组的全部注单（含单关 + 其他串关子单）按同一结果结算；跨组注单独立处理。这意味着多组场景下不需要在串关展开面板内再复制多组分区 UI（避免 UX 冗余），跨组批处理统一走盘口卡片入口。

### 7.3.6.2 分组识别规则（系统自动）

系统按每笔注单的"下注时比分"字段自动聚合：

- 下注时比分相同的注单 → 归为同一组
- 组数 = 盘口内不同"下注时比分"的数量

| 组数 | 场景占比 | 典型触发 |
|------|---------|---------|
| **1 组** | ~90% | 盘口内无比分变化 / IM 比分变化后关旧挂新盘口 |
| **≥ 2 组** | ~10% | IM 比分变化后对同一盘口 ID 走"暂停-恢复"接新单 |

**结算人员无需手动识别分组**，系统自动判定。

### 7.3.6.5 联动规则

**同盘口注单联动**（对应 settlement 第 6.8 节）：

| 场景 | 作用域 |
|------|-------|
| 1 组 | 同盘口全部注单联动（与现有一致）|
| ≥ 2 组 | **同盘口线 + 同基准分组**内联动；跨组独立 |

**配对联动**（MultiLineTable，对应 第 12.15 节）：

| 场景 | 作用域 |
|------|-------|
| 1 组 | 主队选赢 ⇒ 客队自动=输（与现有一致）|
| ≥ 2 组 | 配对作用于**同一基准分组**内；跨组独立 |

**多组场景示例**：
- 操盘手对组 1 某注单选"主队赢" → 组 1 内全部注单 + 同盘口线客队选项自动联动（客队=输）
- 组 2 的 30 笔注单保持待结算，需单独处理
- 操盘手对组 2 选"主队输" → 组 2 内全部注单 + 客队选项联动（客队=赢）

## 7.5 卡片内嵌结算日志

每张卡片底部可展开的日志区域，展示该盘口的结算操作记录：

| 字段 | 说明 |

## 7.7 操作按钮状态

### 7.7.1 可操盘盘口

| 盘口状态 | 结算按钮 | 二次结算按钮 |

### 7.7.2 数据源盘口

| 盘口状态 | 结算按钮 | 二次结算按钮 |
|---------|---------|------------|
| 数据源接管态（任意状态） | 不显示（只读） | 不显示 |
| 待手动结算（超时解锁） | 启用（本地结算） | 不显示 |
| 本地锁定（已本地结算） | 不显示 | 显示「二次结算」按钮（本地差额处理） |

> 盘口状态完整定义和流转规则见[第 18 章](/settlement-detail/18-状态流转规则)。
>
> **赛事级前置限制（精细化，对齐 [18.2.3 关键流转规则](/settlement-detail/18-状态流转规则#_18-2-3-关键流转规则) 和 [15.3.1 前置条件](/settlement-detail/15-驳回与二次结算#_15-3-1-前置条件-首次结算-vs-二次结算区分)）**：
>
> - **首次结算（关盘→已结算）/ 盘口级取消**：赛事状态 = **待结算 或 异常态**允许（行业做法：异常态下有明确赛果的盘口可正常首次结算，如腰斩时上半场类盘口）；已结算 / 已取消时禁用
> - **盘口二次结算（已结算→关盘）**：仅赛事状态 = 待结算；其他状态禁用，鼠标悬停提示"请先执行赛事驳回"
>
> 数据源盘口的按钮展示不受此限制影响（本来基本只读）。
>
> **按钮粒度硬约束**：结算 / 二次结算 / 取消等所有操作按钮均在盘口线级（MultiLineTable）或玩法级（其余渲染器）粒度上展示。**选项级（Matrix 单元格、LongList 单选项、SingleLineTable 选项行、BT6/BT158/BT9 矩阵任一格子）不得出现以上任何按钮**。完整粒度规则见 [第 18 章 18.2.2 操作按钮粒度规则](/settlement-detail/18-状态流转规则#18-2-2)。

## 7.8 盘口结算前置条件

**只要满足以下两个条件，盘口即可执行结算，不需要等整个赛事完赛**：

| 条件 | 说明 |
|------|------|
| 盘口必须已关盘 | 关盘来源：**数据源推送关盘（唯一来源）**。操盘页与结算页面均不提供关盘入口；关盘 = 绝对终态，无人工开盘入口。如数据源误推送关盘，依赖数据源再次推送开盘信号自动响应。未关盘时不展示结算操作（标记"开盘中"）。详见[第 18 章关盘来源](/settlement-detail/18-状态流转规则) |
| 该盘口所需赛果已确定 | 不同判定时机类型的盘口，所需赛果字段不同，详见 7.8.1 |
| **BT6 波胆除外** | BT6 不需要关盘，赛中比分变化时系统实时扫描已无法中奖的选项标记为输（只判输不判赢，判赢需等最终比分确定后关盘）|

### 7.8.1 盘口按判定时机分类

| 判定时机类型 | 所需赛果 | 典型 BT | 结算时机 |
|-----------|---------|---------|---------|
| **上半场类 (1H)** | 半场比分 + 半场扩展（角球/牌）+ 半场进球详情 | BT1-1H / BT2-1H / BT6-1H / BT9 半全场（半场部分）/ 半场角球类 / 半场牌类 | 半场结束后（上半场弹窗应用时）|
| **下半场类 (2H)**【独立分类】| 下半场比分 + 下半场扩展（角球/牌）+ 下半场进球详情 | BT1-2H / BT2-2H / BT6-2H / BT24 / BT25 / BT26 / BT27 / BT28 / BT29 / BT30 / BT33 / BT34 / BT57 / BT19 / BT20 / BT291 等 / 下半场角球类 / 下半场牌类 | 下半场结束后（下半场弹窗应用时，前置上半场已应用）|
| 15 分钟段类 | 段内比分 | BT48 / BT179 / BT180（15 分钟段）| 该段结束后 |
| **事件触发类** | 进球明细 / 角球明细 / 首个事件时间轴 | BT37 第 X 粒进球队伍 / BT159 角球第 X 粒 | **事件发生即可结算**，可能早于终场 |
| 全场类 (FT) | 全场合计比分（= 上+下）+ 全场累计扩展 + 加时 + 点球大战 | BT1 / BT2 / BT3 / BT5 / BT7 / BT8 / BT158 / BT160 / BT161 等 / 全场角球类 / 全场牌类 / BT9 半全场（全场部分）/ 加时类（分区 B）/ 晋级类（分区 C）| 全场结束后（全场扩展弹窗分区 A / B / C 各自应用时）|
| 比分变动触发类 | 赛中实时比分 | BT6 波胆（赛中判输部分）| 比分变化时触发 |

> **事件触发类盘口的特殊性**：这类盘口的"所需赛果"是赛中的具体事件（如第 X 粒进球队伍、第 X 粒角球由哪方发出等），事件发生时数据源返回相应字段，该盘口即可关盘并结算，**完全不依赖终场比分**。这是常规赛事内的特殊判定时机盘口，**不是冠军类长期盘口**（冠军盘口见独立模块 [settlement-outright](/settlement-outright/)）。

> **关于下半场 (2H) 类的独立分类**：
> - 下半场比分本期改为**独立录入**（下半场弹窗录入；数据源开关 = 开时预填 = 数据源全场 − 半场，结算人员核对）；不再采"派生到全场类"方案
> - 判定时机独立：下半场弹窗应用后 2H 类盘口立即可结算，**不依赖全场扩展弹窗**
> - 全场合计比分 = 上半场 + 下半场（只读合计）→ 在全场扩展弹窗分区 A 展示，用于 FT 类盘口结算
> - 从**时段比分维度**（[第 6.2 节 盘口筛选工具栏](./06-盘口筛选与Tab#_6-2-盘口筛选工具栏)），下半场单独一个「**下半场 (2H)**」tab，便于筛选下半场相关玩法

### 7.8.2 按判定时机独立结算

各盘口按自身判定条件走独立结算流程——半场结束后上半场类盘口可结算，第 3 粒进球发生后相关事件触发类盘口可结算，不必等到全场终场。
>
> **关于"赛事完赛"**：数据源在赛果数据中**提供比赛结束时间字段**（仅用于展示），但不单独提供"赛事完赛状态枚举"。赛事级"已结算"由结算人员在所有盘口完成结算后主动点击「结算赛事」产生。
