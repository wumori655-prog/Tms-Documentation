# 第十一章 赛果数据录入规则

> **关盘口径（2026-04-21 生效）**：关盘来源统一为数据源推送（唯一来源）；关盘 = 绝对终态，操盘页与结算详情页均不提供人工开盘入口。

## 11.0 录入层级

**一期仅赛事级录入**（无玩法级特殊录入）：

| 层级 | 触发入口 | 录入范围 | 影响盘口 |
|------|---------|---------|---------|
| **赛事级赛果录入**（按时段 4 个独立入口）| **上半场赛果录入**（上半场类分组顶部，蓝）/ **下半场赛果录入**（下半场类分组顶部，青绿）/ **15 分钟段赛果录入**（15 分钟段类分组顶部，紫，弹窗含 6 Tab）/ **全场赛果与扩展**（全场类分组顶部，绿，弹窗内 3 分区：全场累计扩展 · 加时赛 · 点球大战，各分区独立应用）→ 各自弹窗见[第 9 章 9.1](./09-弹窗交互#_9-1-赛果录入弹窗-4-个独立弹窗) | 按时段分治：半场比分 / 下半场比分 / 各段比分 / 加时 / 点球大战 / 扩展统计 等 | 所有关联盘口（按 [11.6 映射表](#_11-6-赛果数据-盘口映射表) 自动分发到各 BT）|

> **BT158 反波胆说明**：一期 BT158 统一采用赛事级终场比分自动判定（选项结构 5×5 + 其他 = 26 个，与 BT6 取反），**无需玩法级特殊录入**。详见[第 7 章 7.3.3](./07-盘口结算卡片#_7-3-3-matrix-波胆-反波胆-半全场) 和 [第 12 章 12.8](./12-BT级结算判定规则#_12-8-反波胆-bt158)。
>
> **关于"玩法级特殊录入"的预留**：未来若出现赛事级数据无法完全描述结算条件的极少数玩法，可在此处定义玩法级卡片内嵌录入区；一期暂无此类场景。

## 11.1 录入模式

| 模式 | 触发条件 | 录入方式 | 可编辑 | 本期是否实装 |
|------|---------|---------|--------|-------------|
| **数据源同步** | 数据源开关 = 开 + 数据源已拉取该赛事该时段的比分数据 | 系统预填数据源已拉取的比分数据，页面只读核对；结算人员手动点击"应用赛果"触发后续流程 | 否 | **是** |
| **人工录入** | 数据源开关 = 关 或 数据源尚未提供该字段 | 结算人员手动录入 / 修正字段 → 手动点击"应用赛果" | 是 | **是** |
| **混合模式** | 同一赛事部分字段由数据源提供、部分字段缺失 | 基础字段取数据源值（只读核对），缺失字段由结算人员补充；统一手动点击"应用赛果" | 部分 | **是** |

> 赛果字段可按阶段分步录入 / 分步同步（半场字段在半场结束后同步 / 录入；全场字段在全场结束后同步 / 录入），无需等待"赛事完赛"这一统一时刻。

## 11.2 录入时机与前置条件

赛果录入允许**按阶段分步**开启。各时段（半场、15 分钟段、全场）的赛果由数据源分别同步或由结算人员分别录入，对应时段的盘口一旦关盘即可走结算流程。

> 数据源会在完整赛果数据中返回"比赛结束时间"字段，但这只是赛果数据里的一个时间戳，不是业务规则的前置条件——盘口结算只需盘口关盘 + 该盘口所需赛果字段到位即可，不要求先拿到"比赛结束时间"。

### 11.2.1 前置条件（分层独立）

| 层级 | 前置条件 | 未满足时行为 |
|------|---------|-----------|
| 赛事级赛果录入弹窗（3 个按时段独立弹窗）| **数据源开关 = 开**时：对应时段盘口已关盘 或 数据源已拉取该时段比分数据；**数据源开关 = 关**时：直接可开启（无需数据源前置）| 数据源开关 = 开且条件未满足：对应时段按钮灰置（半场/分段/全场按钮各自启用条件，见 [9.1.0](./09-弹窗交互#_9-1-0-弹窗总览-3-个按钮-3-个弹窗)）|
| 盘口级结算 | 盘口已关盘 + 该盘口所需赛果字段已确定（数据源同步预填 或 人工录入，两种来源均可）| 盘口卡片标记「开盘中」或「缺少数据」；结算按钮禁用 |
| 赛事级结算赛事 | 该赛事下所有盘口均已结算或已取消 | 「结算赛事」按钮禁用，展示未完成盘口清单 |

### 11.2.2 弹窗字段归属矩阵（4 弹窗分治 + 全场扩展 3 分区）

##### 分区 C：点球大战（赛事进入点球大战才启用）

| 字段 | 必填/选填 | 说明 |
|------|---------|------|

**前置条件**：分区 B 已应用 且 加时全场平局（否则点球大战不启用）。

**独立字段**：点球大战比分**不计入**任何全场 / 加时比分字段，仅用于决定晋级方。

#### 11.2.2.5 关键字段定义

- **半场比分**：常规 45 分钟 + 半场伤停的主/客得分
- **下半场比分**：46-90 分钟 + 全场伤停的主/客得分（独立录入，不派生）
- **全场比分**：= 半场 + 下半场 自动合计（只读展示），**不含加时与点球大战**
- **加时比分**：加时上（91-105' + 伤停）+ 加时下（106-120' + 伤停）；加时全场由系统自动合计
- **点球大战比分**：加时结束仍平局后独立进行，**独立字段**，仅用于决定晋级方
- **扩展统计（角球 / 牌）按时段独立录入**：半场在上半场弹窗录；下半场在下半场弹窗录；加时段在全场扩展弹窗加时区录；全场累计值由系统自动合计（只读）
- **触发启用**：加时分区仅当赛事实际进入加时阶段时启用；点球大战分区仅当赛事进入点球大战阶段时启用；常规 90 分钟结束的赛事加时/点球分区保持禁用，对应盘口按"未进入"规则处理（走水/取消视盘口规则而定）

### 11.2.3 操作流程

```
各时段赛果分阶段可得（半场结束、全场结束、15 分钟段结束等）
  ↓
数据源自动拉取该阶段字段（数据源开关 = 开时）或 结算人员手动打开「赛果录入」弹窗
  ↓
结算人员打开对应时段的赛果录入弹窗（4 弹窗择一：上半场 / 下半场 / 15 分钟段 / 全场扩展）
  · 数据源同步模式：已拉取的字段预填（只读核对）
  · 人工录入模式：所有字段可编辑
  · 混合模式：数据源已拉取字段只读，缺失字段可编辑
  ↓
结算人员核对 / 补充 / 修正字段
  ↓
系统实时校验（弹窗内 + 跨弹窗一致性，见 11.4 节）
  ↓ 校验通过
结算人员点击「应用赛果」（本期统一手动点击，不自动触发）
  ↓
系统对"所需赛果字段已确定 + 已关盘"的盘口按 BT 级判定规则**预填**结算结果到盘口卡片
  ↓
结算人员逐盘口确认（点击盘口卡片的「结算」按钮，本期统一手动确认）
  ↓
所有盘口结算完成后 → 结算人员点击「结算赛事」
```

### 11.2.4 IM 推送比分数据拉取与兜底

| 规则 | 说明 |
|------|------|
| 拉取时机 | IM 推送比分数据后即可拉取；支持分阶段（半场数据、分段数据可先于全场提供）|
| 已拉取 | 允许打开赛果录入弹窗，并开放后续结算 |
| 拉取中/失败 | 不允许录入和结算，仅展示等待/重试状态 |
| 盘口终态缺失 | 已拉到比分数据但 IM 未推送盘口终态时，允许人工补录并由本地系统按 BT 级判定规则计算 |
| 混合模式 | 基础比分以已拉取赛果为准，缺失字段允许人工补充 |
| 自收结算兜底 | IM 盘口终态推送未送达时，本地系统根据已拉取比分数据 + 注单下注时比分按规则计算结算结果（按 12.0.2 下注后比分口径）|

## 11.3 录入数据分层

> **必填/选填 的判定口径**：
> - "**必填**" = 该赛事**必须录入**，未录入时相关结算全部阻塞
> - "**条件必填（有依赖盘口时必填）**" = 本赛事存在依赖该字段的盘口时**必须录入**（否则盘口结算按钮禁用、展示"缺少数据"，见 第 11.5 节）；本赛事不存在依赖盘口时可跳过
> - "**条件必填（赛事进入加时/点球大战）**" = 赛事实际进入对应阶段时必须录入；否则该字段不启用
> - "**自动**" = 系统自动计算，结算人员不操作

| 层级 | 数据项 | 必填/选填 | 输入控件 | 依赖玩法 | 所属弹窗 / 分区 |
|------|--------|---------|---------|---------|---------------|
| 基础层 | 半场比分（主/客）| 必填 | 两个整数输入框 | 1H 类 + BT9 半场部分 | **上半场弹窗** |
| 基础层 | 下半场比分（主/客）| 必填（数据源开关 = 开时预填 = 全场 − 半场）| 两个整数输入框 | 2H 类全部 | **下半场弹窗** |
| 基础层 | 全场比分（主/客，只读合计 = 半场 + 下半场）| 自动 | 只读 | FT 类全部 | **全场扩展弹窗 · 分区 A** |
| 基础层 | 总进球 / 单双结果 | 自动计算 | 只读 | BT7 / BT5 | **全场扩展弹窗 · 分区 A（派生）** |
| 进球层 | 上半场进球详情（时间 ≤ 45'+ 伤停）| 条件必填（有 BT159 / BT37 依赖时）| 动态行表格 | BT159 / BT37 半场相关 | **上半场弹窗** |

## 11.4 校验规则（按弹窗分治）

校验在录入过程中实时执行。校验失败时，对应输入框红色高亮 + 下方展示提示文案，当前弹窗的「应用赛果」按钮禁用。

#### 11.4.4.A 分区 A：全场累计扩展

#### 11.4.4.B 分区 B：加时赛（赛事进入加时才启用）

| 加时段进球归属一致 | 加时进球详情行数 = 加时全场总进球（主+客）| 「加时进球详情数量与比分不一致」|

### 11.4.5 跨弹窗一致性校验（完整清单，阻塞式）

**触发时机**：弹窗 / 分区应用赛果时。任一校验失败，阻塞当前应用按钮，不允许矛盾赛果进入系统。

| 跨弹窗校验 | 触发方 | 规则 | 失败提示 |
|----------|-------|------|---------|
| **上 + 下 合计非负校验** | 下半场应用时 | 下半场主 ≥ 0 且 下半场客 ≥ 0 | 「下半场比分不能为负」|
| **上 + 下 = 数据源全场比分（核对）**| 下半场应用时（若数据源已推送全场比分）| 半场主 + 下半场主 = 数据源全场主；客队同理 | 「上半场 + 下半场 合计 X:Y 与数据源全场 A:B 不一致，请核对」|
| **全场合计 ≥ 半场** | 自动（分区 A 合计值展示时）| 合计值约束（主客各自 ≥ 半场）| 合计派生，天然满足 |
| **段 1+2+3 = 半场** | 第 3 段应用 / 上半场应用（互相触发）| 段 1+2+3 主 = 半场主 · 段 1+2+3 客 = 半场客 | 「段 1-3 累计 A:B 与半场比分 X:Y 不一致」|
| **段 4+5+6 = 下半场** | 第 6 段应用 / 下半场应用（互相触发）| 段 4+5+6 主 = 下半场主 · 段 4+5+6 客 = 下半场客 | 「段 4-6 累计 A:B 与下半场比分 X:Y 不一致」|
| **段 1-6 累计 = 全场合计** | 任一段 / 分区 A 应用时 | 所有段累计 = 半场 + 下半场 | 「6 段累计 A:B 与全场合计 X:Y 不一致」|
| **段累计单调不减** | 每段 Tab 应用时 | Σ(段 1-N) 主客累计随 N 单调不减 | 「该段累计比分小于前一段」|
| **半场角球 + 下半场角球 = 全场合计角球** | 分区 A 应用时 | 半场角球 + 下半场角球 = 合计值（自动派生，校验一致）| 自动派生天然一致；若有加时段角球 → 合计 = 半场 + 下半场 + 加时段 |
| **半场牌 + 下半场牌 = 全场合计牌** | 同上 | 同上 | 同上 |
| **进球详情时间分布一致** | 分区 A 应用时 | 上半场进球数（上半场弹窗）+ 下半场进球数（下半场弹窗）= 全场合计总进球 | 「进球详情总数与全场比分不一致」|
| **进球详情归属与段一致** | 15 分钟段应用时（若上/下半场已应用）| 段 1-3 进球归属 = 上半场进球详情；段 4-6 进球归属 = 下半场进球详情 | 「段内进球与上/下半场进球明细不一致」|
| **加时下 ≥ 加时上（累计）**| 分区 B 应用时 | 加时下累计 ≥ 加时上 | 「加时下比分应大于等于加时上」|
| **加时段进球数 = 加时全场总进球** | 分区 B 应用时 | 加时进球详情行数 = 加时全场主 + 客 | 「加时进球详情数量与加时比分不一致」|

**阻塞行为**：
- 校验失败时，当前应用按钮所在的弹窗 / 分区顶部展示红色横幅提示 + 定位到失败字段高亮
- 对应的「应用」按钮禁用
- 不允许绕过（即使结算人员坚持，也需先修正源弹窗或调整本弹窗）

**修正路径**：
- 若上半场已应用且下半场录入与其合计矛盾 → 结算人员须先判断哪边正确
  - 若上半场错 → 需先对 1H 类已结算盘口执行二次结算并重录上半场
  - 若下半场录入错 → 当前弹窗内直接修正
- 若下半场已应用且上半场录入与其矛盾（反向校验）→ 同上
- 全场扩展分区与上/下半场不一致 → 修正上/下半场或调整全场扩展字段

### 11.4.6 校验时机与行为

| 规则 | 说明 |
|------|------|
| 实时校验 | 输入过程中实时校验，输入框失去焦点时触发 |
| 红色高亮 | 校验失败的输入框边框变红 |
| 提示文案 | 输入框下方展示红色提示文案 |
| 按钮禁用 | 任一校验失败时，当前弹窗 / 分区的「应用」按钮禁用 |
| 多条同时失败 | 所有失败项同时高亮并展示提示 |
| 修正后自动恢复 | 修正后校验通过，红色高亮消失，按钮恢复可用 |
| 跨弹窗校验触发时机 | 仅在应用按钮点击时校验（不在输入过程中做跨弹窗比对，避免过度打扰录入）|
| 分区应用按钮独立 | 全场扩展弹窗内分区 A / B / C 各有独立应用按钮；分区之间按前置硬约束依次启用（A → B → C）|

## 11.5 数据缺失时的盘口处理（按弹窗分治）

| 场景 | 受影响盘口 | 盘口卡片表现 | 提示文案 |

## 11.6 赛果数据 → 盘口映射表

录入每项赛果数据后，系统自动计算以下盘口的结算结果。开发必须按此映射实现。

### 全场比分（主 X : 客 Y）

| 影响的盘口 | 判定依据 |
|-----------|---------|
| BT1 全场让球 | 净差值 = X - Y + 盘口线 |
| BT2 全场大小 | 净差值 = (X+Y) - 盘口线 |
| BT3 全场独赢 | X vs Y 直接比较 |
| BT5 全场单双 | (X+Y) 奇偶 |
| BT6 全场波胆 | 精确匹配 X:Y |
| BT7 总进球 | X+Y 匹配选项 |
| BT8 双重机会 | X vs Y → 映射 1X/12/X2 |
| BT18 双方进球 | X ≥ 1 且 Y ≥ 1 |
| BT22 主队零失球 | Y = 0 |
| BT23 客队零失球 | X = 0 |
| BT158 反波胆 | X:Y ≠ 选项比分 |
| BT160 主队大小 | 净差值 = X - 盘口线 |
| BT161 客队大小 | 净差值 = Y - 盘口线 |

### 半场比分（主 A : 客 B）

| 影响的盘口 | 判定依据 |
|-----------|---------|
| BT1-1H 上半场让球 | 净差值 = A - B + 盘口线 |
| BT2-1H 上半场大小 | 净差值 = (A+B) - 盘口线 |
| BT6-1H 上半场波胆 | 精确匹配 A:B |
| BT9 半全场 | 半场结果(A vs B) + 全场结果(X vs Y) 组合 |

### 系统自动计算 2H 下半场

| 自动计算 | 公式 | 影响的盘口 |
|---------|------|-----------|
| 2H 主队 | X - A | BT1-2H、BT2-2H、BT6-2H |
| 2H 客队 | Y - B | 同上 |

### 15分钟段比分

| 影响的盘口 | 判定依据 |
|-----------|---------|
| BT48 15分钟1X2 | 该段主客比分差 |
| BT179 15分钟让球 | 段内净差值 = 段内主队 - 段内客队 + 盘口线 |
| BT180 15分钟大小 | 段内净差值 = 段内总进球 - 盘口线 |

### 进球详情

| 影响的盘口 | 判定依据 |
|-----------|---------|
| BT159 第X粒入球球队 | 第X粒进球由哪队打入 |

### 扩展统计

| 数据 | 影响的盘口 |
|------|-----------|
| 角球数（主/客） | 角球让球、角球大小（数据源盘口） |
| 红黄牌数 | 红牌大小、黄牌大小（数据源盘口） |

## 11.7 录入后的系统行为

结算人员完成某弹窗录入并点"应用赛果"后：

| 步骤 | 说明 |
|------|------|
| 1 | 保存本次弹窗的赛果数据 |
| 2 | 系统对比本弹窗字段是否发生变化（与上次应用值比较）|
| 3 | **按弹窗 / 分区独立判断影响范围**：上半场弹窗变化 → 影响 1H 类 + BT9 半场部分 + 半场角球/牌类 + BT159/BT37 半场相关；下半场弹窗变化 → 影响 2H 类全部 + 下半场角球/牌类 + BT159/BT37 下半场相关；15 分钟段 Tab 变化 → 影响该段对应 BT；全场扩展分区 A → FT 类全部 + BT9 全场部分 + 全场角球/牌累计类；分区 B → 加时类 + 加时角球/牌；分区 C → 晋级类 |
| 4 | **仅对受影响且已关盘的盘口**，根据映射表（11.6）重新计算结算结果 |
| 4a | **含多组基准分的盘口**（详见[第 7.3.6 节](./07-盘口结算卡片#_7-3-6-自适应基准分组结算规则-核心规则)）：系统按组独立推导基准分（每组自取其下注时比分代入终场比分 / 半场比分 / 分段比分公式），分别预填每组结果；多组弹窗 UX 走 [§7.3.6.4 首次结算弹窗分区](./07-盘口结算卡片#_7-3-6-4-弹窗-ux-示意)，结算人员按组分别核对 / 调整 / 确认 |
| 5 | 赛果未变化或本弹窗未受影响的盘口不重新计算，保持原结算结果 |
| 6 | 变化的盘口卡片展示新的预填结算结果（结算人员可修改）；多组盘口按组分区展示各组的预填结果 |
| 7 | 结算人员逐盘口确认（点击「结算」按钮执行）；多组盘口需按组分别确认（可一次性全部选定后合并提交，也可逐组提交）|
| 8 | 本次应用后的赛果数据锁定，**本弹窗不可再次应用**；如需修改需对受影响的已结算盘口先执行二次结算（多组盘口支持"全部组统一二次结算"或"仅二次结算某一组"，见[第 9.4.3 节](./09-弹窗交互#_9-4-3-多组基准分盘口的自适应展示-对齐-第-7-3-6-节)） |

### 自动结算触发规则

| 规则 | 说明 |
|------|------|
| 触发条件 | 赛果数据发生变化（比分变了、进球详情变了等） |
| 不触发 | 赛果数据未变化时，不重新计算，不覆盖已有结算结果 |
| 已关盘盘口（赛事**待结算**，盘口待结算） | 计算结算结果，预填到盘口卡片 |
| 已结算盘口（赛事**待结算**） | 盘口赛果改变时，系统自动重新计算并执行二次结算（等效于自动执行"二次结算 → 重算 → 再次结算"的合并操作），无需结算人员手动干预；按 15.4 差额矩阵自动处理派彩差额。若赛果未变（例如重选相同结果），则不触发任何注单级联 |
| 开盘中盘口 | 不计算结算结果（BT6 波胆比分变动触发结算除外） |

### 关于"关盘"操作

| 场景 | 行为 |
|------|------|
| 盘口已关盘 | 不会再次执行关盘操作，仅应用赛果数据并计算结算结果 |
| 盘口仍在开盘 | 不自动关盘，不计算结算结果。盘口需通过数据源 IM 推送关盘后，才进入可结算状态 |

## 11.8 边界场景与异常处理

> **前置说明**：以下「比分拉取中」「拉取失败」相关场景**仅在数据源开关 = 开**时成立；数据源开关 = 关时，赛果录入**略过拉取**（无请求、无 loading 态、无拉取中提示），所有字段默认可编辑。

| 场景 | 适用条件 | 系统行为 | 提示文案 |
|------|---------|---------|---------|
| 打开赛果录入时系统正在拉取比分 | 数据源开关 = 开 | 弹窗展示 loading 覆盖层 + 所有比分输入框 disabled；拉取完成后自动回填 | 「⏳ 比分拉取中，请稍候……」 |
| 比分拉取超时（如 > 10s 未返回） | 数据源开关 = 开 | 输入框保持 disabled；展示重试按钮 | 「比分拉取超时，请点击重试」 |
| 比分拉取失败（网络/IM 异常） | 数据源开关 = 开 | 输入框保持 disabled；展示重试按钮 + 错误码 | 「比分拉取失败（IM-503），请重试或联系运维」 |
| 数据源开关 = 关时打开赛果录入 | 数据源开关 = 关 | 略过拉取；所有比分输入框直接可编辑（人工录入模式）| — |
| 比分输入为空后点击应用 | 所有模式 | 按钮禁用 | 「请输入比分」 |
| 输入非数字字符 | 输入框不接受，自动过滤 | — |
| 输入超大数值（如 999） | 允许输入但展示橙色提示 | 「比分异常偏高，请确认是否正确」 |
| 半场比分 > 全场比分 | 红色高亮 + 按钮禁用 | 「半场比分不能大于全场比分」 |
| 进球详情与比分不一致 | 红色高亮 + 按钮禁用 | 「进球详情与总进球不一致」 |
| 已应用赛果后尝试修改 | 输入框禁用 | 「赛果已应用，如需修改请先二次结算」 |
| 网络中断时提交 | 提交失败，保留已输入数据 | 「网络异常，请重试」 |
| 数据源开关 = 开 且 IM 推送新比分时本地正在录入（见下 IM 覆盖边界细分）| 按场景处理 | 按场景提示 |
| 数据源开关 = 关 时 IM 推送新比分 | 不自动同步，保留本地录入 | —（系统不展示提示） |
| 赛事驳回后对已结算盘口执行二次结算 | 该盘口赛果录入解锁，恢复可编辑 | 「盘口二次结算中，赛果可重录」 |
| 录入赛果但盘口仍在开盘 | 不计算结算结果，不关盘 | 「该盘口仍在开盘中，结算需等待关盘」 |
| 赛果未变化时重复提交 | 不重新计算，保持原结果 | 「赛果未发生变化，无需重新计算」 |
| 赛果变化导致已结算盘口结果改变 | 系统直接二次结算 + 自动差额处理 | 「赛果变化，盘口 {ID} 已二次结算，差额 ¥{N} 已处理」 |
| **下半场弹窗打开但上半场未应用** | 前置硬约束阻塞，下半场按钮灰置 | 「请先录入并应用上半场赛果」|
| **上+下 合计 ≠ 数据源全场比分（数据源已推送时）**| 下半场应用时橙色提示（不阻塞）| 「上 + 下 合计 X:Y 与数据源全场 A:B 不一致，请核对」|
| **段 1+2+3 ≠ 半场** 或 **段 4+5+6 ≠ 下半场** | 阻塞末次应用 | 「段累计比分与对应半场比分不一致」|
| **段 1-6 累计 ≠ 全场合计** | 阻塞末次应用 | 「6 段累计比分与全场合计不一致」|
| **2H 类盘口结算时下半场未应用** | 显示缺少数据 | 「2H 类盘口需下半场赛果已应用」|
| **FT 类盘口结算时全场扩展 · 分区 A 未应用** | 显示缺少数据 | 「FT 类盘口需全场扩展 · 分区 A 已应用」|
| **全场扩展分区 A 打开时上半场或下半场未应用** | 前置硬约束阻塞 | 「请先应用上半场 + 下半场赛果」|
| **全场扩展分区 B 打开时分区 A 未应用** | 前置硬约束阻塞 | 「请先应用全场扩展 · 分区 A」|
| **全场扩展分区 C 打开时分区 B 未应用** | 前置硬约束阻塞 | 「请先应用加时赛」|

### 11.8.1 IM 覆盖本地录入的边界细分（数据源开关 = 开时）

IM 推送新比分数据与本地录入冲突时，按结算人员所处的操作阶段细分处理：

| 操作阶段 | 系统行为 | 提示文案 |
|---------|---------|---------|
| 阶段 1：结算人员**正在录入中**（未点"应用赛果"）| **数据源覆盖本地录入**，页面字段刷新为数据源值；本地未保存的输入丢失。**未保存即未录入**；比分一旦点击"应用赛果"即进入阶段 2/3 的保护机制 | 「数据源已更新赛果，页面已刷新」|
| 阶段 2：录入完成、未点"应用赛果"（字段已填但弹窗未提交）| 弹窗保留 + **顶部弹出选择提示**：「数据源已更新赛果，是否用数据源值替换？」两选项：「使用数据源值」/ 「保留我的录入」| 「数据源已更新赛果，请选择如何处理」|
| 阶段 3：赛果**已应用但盘口未结算**（盘口处于"关盘 + 待结算"）| IM 覆盖 + 预填结算结果自动刷新；弹窗关闭状态下不打扰结算人员，重新打开弹窗时展示新赛果 | 「数据源更新赛果，盘口 {ID} 预填结果已刷新」|
| 阶段 5：赛果已应用且**盘口已结算**（赛事已结算 / 已取消 / 异常）| 走 第 11.7 节 "仅告警不动数据"规则，弹窗顶部提示 | 「比分已更新，如需修正请先执行赛事驳回」|

**扩展数据的保留策略**（进球详情 / 角球 / 红黄牌等）：

| IM 推送的覆盖范围 | 扩展数据处理 |
|-----------------|------------|
| IM 推送的新比分数据**包含扩展字段**（如新进球详情）| 按阶段规则覆盖扩展字段 |
| IM 推送的新比分数据**不包含扩展字段** | **保留本地已录入的扩展数据**，不清空（避免人工录入的角球/红黄牌等被意外清除）|
