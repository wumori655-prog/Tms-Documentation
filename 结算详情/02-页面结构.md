# 第二章 页面结构

> **关盘口径（2026-04-21 生效）**：关盘来源统一为数据源推送（唯一来源）；关盘 = 绝对终态，操盘页与结算详情页均不提供人工开盘入口。

## 2.1 布局设计

结算详情页采用**三栏布局**：左侧赛事切换 + 中间操作区 + 右侧日志面板。

```
┌──────────────────────────────────────────────────────────────────────┐
│ 顶部导航栏（告警摘要 + 数据源状态 + 刷新）                                       │
├──────────┬──────────────────────────────────┬───────────────────────┤
│          │ 赛事信息头（比分/状态/人员/操作按钮） │                       │
│ 左侧     │────────────────────────────────────│ 右侧                  │
│ 赛事切换栏 │ 盘口数据汇总（本地/数据源/合计）     │ 日志面板               │
│          │────────────────────────────────────│                       │
│ 待结算赛事 │ [可操盘盘口] [数据源盘口] Tab       │ 搜索 + 筛选            │
│ 快捷列表   │────────────────────────────────────│                       │
│          │ 筛选工具栏（阶段/时段/状态/搜索）    │ 日志时间线             │
│ 搜索      │────────────────────────────────────│                       │
│          │ ┌【上半场类】[📝 上半场赛果录入]──┐    │                       │
│ 状态图例   │ │ BT1-1H / BT2-1H / BT9 ... │      │                       │
│          │ └──────────────────────────┘      │                       │
│          │ ┌【下半场类】[📝 下半场赛果录入]──┐    │                       │
│          │ │ BT1-2H / BT2-2H / BT6-2H... │    │                       │
│          │ └──────────────────────────┘      │                       │
│          │ ┌【15 分钟段类】[📝 15 分钟段赛果录入]┐   │                       │
│          │ │ BT48 / BT179 / BT180       │    │                       │
│          │ └──────────────────────────┘      │                       │
│          │ ┌【事件触发类】（无独立录入按钮）┐   │                       │
│          │ │ BT37 / BT159              │     │                       │
│          │ └──────────────────────────┘      │                       │
│          │ ┌【全场类】[📝 全场赛果与扩展]──┐    │                       │
│          │ │ BT1 / BT2 / BT3 / BT158... │    │                       │
│          │ └──────────────────────────┘      │                       │

│          │ │ BT6 波胆 ...              │     │                       │
│          │ └──────────────────────────┘      │                       │
└──────────┴──────────────────────────────────┴───────────────────────┘
```

> **赛果录入 UI 结构**：
> - 顶部无统一「赛果录入」按钮
> - **按时段分组**，每个时段分组顶部有**独立的赛果录入按钮**（4 个按钮：上半场 / 下半场 / 15 分钟段 / 全场扩展）
> - **15 分钟段按钮**点击后打开一个弹窗，弹窗内含 6 个 Tab（对应 6 个 15 分钟段），每个 Tab **独立应用**，仅影响该段盘口（不影响其他 Tab 和其他页面）
> - **全场赛果与扩展按钮**点击后打开一个弹窗，弹窗内含 3 个分区（全场累计扩展 · 加时赛 · 点球大战），**各分区独立应用按钮**，按前置硬约束依次启用（A → B → C）
> - **事件触发类**（BT37/BT159）依赖进球 / 角球明细——进球明细在上半场 / 下半场 / 全场扩展弹窗的"进球明细"子模块按时段分别录入（事件发生时即可结算，不依赖终场）
> - **BT6 波胆赛中判输** 依赖实时比分推送，无需独立录入按钮（比分变化自动触发判输扫描）
>
> 详见[第 7 章盘口结算卡片](./07-盘口结算卡片)按时段分组规则、[第 9 章 9.1 赛果录入弹窗](./09-弹窗交互) 弹窗结构。

## 2.3 模块职责一览

| 模块 | 位置 | 职责 | 章节 |
|------|------|------|------|
| 左侧赛事切换栏 | 左侧 | 待结算赛事快捷切换 | [第 3 章](./03-左侧赛事切换栏) |
| 赛事信息头 | 中间顶部 | 比分（隐含比分拉取状态）、比赛状态、人员、操作按钮（按赛事状态**互斥**显示：结算赛事 / 赛事驳回 / 赛事异常处理，同一时刻只显示其中之一）| [第 4 章](./04-赛事信息头) |
| 盘口数据汇总 | 中间 | 本地 / 数据源 / 合计的注单数、投注额、派彩、盈亏 | [第 5 章](./05-盘口数据汇总) |
| 盘口筛选与 Tab | 中间 | 本地 / 数据源 Tab 切换 + 筛选 | [第 6 章](./06-盘口筛选与Tab) |
| 盘口结算卡片 | 中间 | 4 种渲染器的结算卡片 + 结算操作 | [第 7 章](./07-盘口结算卡片) |
| 数据源盘口 | 中间 | 结算完全依赖 IM 驱动的盘口，默认只读；数据源对该盘口超时未推送盘口终态 + 该盘口所需比分已可得时，自动解锁为「待手动结算」；本地结算后进入本地锁定态 | [第 8 章](./08-数据源盘口) |
| 弹窗 | 模态层 | 各按钮独立触发的弹窗（**互斥显示，同一时刻只打开一个**）：赛果录入（4 个按时段独立按钮：上半场 / 下半场 / 15 分钟段 / 全场扩展）、结算赛事确认（赛事信息头「结算赛事」按钮）、赛事异常处理（赛事信息头「赛事异常处理」按钮）、赛事驳回（赛事信息头「赛事驳回」按钮）、盘口二次结算（盘口卡片「二次结算」按钮）、结算人员分配（赛事信息头结算人员区域）| [第 9 章](./09-弹窗交互) |
| 右侧日志面板 | 右侧 | 结算日志时间线 | [第 10 章](./10-右侧日志面板) |
