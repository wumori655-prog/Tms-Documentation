# 结算详情页（常规赛事）

> **关盘口径（2026-04-21 生效）**：关盘来源统一为数据源推送（唯一来源）；关盘 = 绝对终态，操盘页与结算详情页均不提供人工开盘入口。如需从关盘恢复，依赖数据源再次推送开盘信号自动响应。

**本页面仅适用于常规单场赛事（EventTypeId = 2）**。冠军赛事 / 长期盘口（EventTypeId = 1，如夺冠预测、晋级预测、最佳球员等）由[冠军赛事结算详情页](/settlement-outright/)独立处理。

单场赛事的结算操作页面，负责赛果录入、盘口级结算执行、赛事异常处理和赛事级确认结算。

**盘口级结算前置**：盘口关盘 + 该盘口所需赛果已确定（可来自数据源或人工录入）。各盘口按自身判定时机独立结算：
- 全场类盘口（BT1 / BT2 / BT3 / BT5 / BT7 / BT8 / BT158 / BT160 / BT161 等）：全场结束后结算
- 上半场类盘口（BT1-1H / BT2-1H / BT9 半全场）：半场结束后结算
- 15 分钟段类盘口（BT48 / BT179 / BT180 15 分钟段）：该段结束后结算
- **事件触发类盘口**（BT37 第 X 粒进球队伍 / BT159 角球第 X 粒）：对应事件发生后即可结算，可能远早于终场
- 比分变动触发类（BT6 波胆赛中判输）：比分变化时实时触发结算（只判输不判赢）

完整分类见[第 7 章 7.8.1](./07-盘口结算卡片#_7-8-1-盘口按判定时机分类)。

**赛果录入触发条件**：至少一个盘口已关盘，或 IM 已推送任一阶段比分数据。不依赖"比赛结束时间"。

**赛事级确认前置**：所有盘口均已结算或已取消。数据源在赛果数据中提供"比赛结束时间"字段（仅作展示），但不单独推送"赛事状态枚举"。赛事级「已结算」由结算人员确认所有盘口完成后主动点击「结算赛事」产生。

## 核心范式

| 要点 | 说明 | 详见 |
|------|------|------|
| 两层状态独立 | 赛事级（待结算 / 已结算 / 已取消 / 异常）与盘口级（开盘 / 关盘 / 已结算 / 已取消）完全独立；两层互不自动联动 | [第 18 章](./18-状态流转规则) |
| 赛果驱动 | 注单级联的唯一触发条件是盘口赛果的实质变化（首次确定或发生改变）；盘口状态变化但赛果未变不触发 | [第 11 章 11.7](./11-赛果数据录入规则)、[第 14 章 14.7](./14-串关结算规则) |
| 盘口来源分离 | 可操盘盘口与数据源盘口走不同结算路径；关盘 = 绝对终态，均不提供人工开盘入口；数据源盘口生命周期由数据源驱动 | [第 8 章](./08-数据源盘口) |
| 数据源本地锁定 | 数据源盘口关盘 + 所需比分数据可得 + 数据源超时未推送盘口终态 → 解锁为待手动结算 → 人工结算后进入本地锁定态（单向不可逆）| [第 8 章 8.3](./08-数据源盘口) |
| 24 小时驳回窗口 | 赛事终态可在距最新结算时间 24 小时内驳回回待结算；超窗后终态永久化 | [第 15 章 15.2](./15-驳回与二次结算) |
| 串关按盘口来源分支 | 可操盘盘口子单本地引擎；数据源盘口子单 IM 有数据走 L1、IM 超时走本地兜底；术语统一用"母单 / 子单" | [第 14 章 14.2b](./14-串关结算规则) |
| 结算人员主动发起 | 所有赛事异常（4 种原因：取消 / 延期 / 腰斩 / 数据错误）均须结算人员通过「赛事异常处理」按钮（仅赛事待结算时启用）主动发起；系统不做定时提醒或超时自动标记；数据源虽提供比分 / 比赛结束时间等结构化字段，但不单独推送"赛事状态枚举" | [第 16 章](./16-赛事异常处理) |
| 赛果录入按时段分组 | **一期仅赛事级录入**（无玩法级特殊录入），按时段分为 4 个独立入口：上半场赛果录入 / 下半场赛果录入 / 15 分钟段赛果录入（弹窗含 6 Tab，每 Tab 独立应用）/ 全场赛果与扩展（弹窗内 3 分区：全场累计扩展 · 加时赛 · 点球大战，各分区独立应用；含进球明细子模块服务事件触发类盘口）。顶部导航栏不再有统一按钮 | [第 9 章 9.1](./09-弹窗交互)、[第 11 章 11.0](./11-赛果数据录入规则) |
| 可操盘盘口自动结算 | 数据源开关 = 开时，本地按 IM 推送的比分数据自动算结果并结算；开关 = 关时需结算人员手动录入比分。数据源盘口独立由 IM 状态字段驱动，超时才人工介入 | [第 1 章 1.5](./01-产品概述) |
| 多组基准分自适应 | 数据源在比分变化后走"暂停—恢复"使同盘口出现多组基准分时，首次结算弹窗按组分区、组内联动、跨组独立；二次结算时差额基线按组独立维护（跨组不串扰），循环计数按组独立累计 | [第 7.3.6 节](./07-盘口结算卡片#_7-3-6-自适应基准分组结算规则-核心规则)、[第 15.4.4 节](./15-驳回与二次结算#_15-4-4-多组基准分场景下的差额基线-组级独立口径) |
| 风控配置数据来源 | 结算模块不管理风控（限额 / 告警 / 赔率边界 / RTP 等配置），所有风控参数**由体育后台下发到操盘后台**，结算场景仅作消费方；本模块中涉及"风控管理配置"的引用指向体育后台下发的参数，结算模块不提供配置入口 | 详见 [第 20 章 配置项汇总](./20-配置项汇总) |
| 结算报表范围 | 本期**不做结算报表模块**；一期结算完成的数据仅在本模块与结算列表内留存并支持审计查询 | 详见本页"关联模块"章节 |
