# 第二十一章 结算日志

> **关盘口径（2026-04-21 生效）**：关盘来源统一为数据源推送（唯一来源）；关盘 = 绝对终态，操盘页与结算详情页均不提供人工开盘入口。

> **规则声明**：本章为结算模块结算日志的**核心规则**。结算列表（[settlement 第 18 章](/settlement/18-结算日志)）遵循本章定义的字段结构与操作类型命名，仅记录其自身入口触发的操作（如人员分配等），核心结算日志格式与本章完全一致。

## 21.1 日志字段结构（核心规则，两端共用）

所有结算日志使用统一字段结构，结算列表和结算详情页共享：

| 字段 | 格式 | 说明 |
|------|------|------|
| 操作时间 | YYYY-MM-DD HH:mm:ss（UTC 时间戳记录，展示按用户本地时区转换）| 精确到秒 |
| 操作人 | 结算人员姓名 / 结算主管姓名 / 「系统」 | 见 21.3 操作人标注规则 |
| 操作人 IP | xxx.xxx.xxx.xxx | 系统自动操作为空 |
| 操作类型 | 见 21.2 操作类型清单 | 枚举写死 |
| 操作对象 | 盘口 ID / 赛事 ID / 母单 ID / 注单 ID | 按操作类型决定 |
| 操作内容 | 变更前 → 变更后 | 自由文本 |
| 操作结果 | 成功 / 失败（含失败原因）| — |
| 关联信息 | 受影响的注单数 / 母单数 / 差额金额 / 累计次数等（若涉及）| 按操作类型决定 |
| 来源页面 | 结算详情页 / 结算列表-注单视图 / 结算列表-赛事视图 / 系统 | 用于跨模块审计 |

> **F9 / F10 / F11 / F12 说明**：
> - **F9 / F10 / F11** = IM 推送触发的自动响应（24h 窗口内），绿色系统级日志
> - **F12** = IM 推送被 24h 超窗规则**拒收**（超窗拒收保护），红色警告级日志，供合规审计
> - 关盘 = 绝对终态，人工无开盘入口；数据源推送开盘信号作为硬约束须自动响应，**但受 24h 窗口约束**：超窗即使数据源再推送也不执行，记 F12 拒收日志
> - 完整处理流程见 [第 15.6d 节](./15-驳回与二次结算#_15-6d-数据源推送开盘信号的自动回滚流程)

## 21.6 与结算列表日志的一致性

结算列表（[settlement 第 18 章](/settlement/18-结算日志)）的日志完全遵循本章定义：
- **字段结构**：完全一致（21.1 为 核心规则）
- **操作类型命名**：完全一致（21.2 为 核心规则）
- **操作人标注规则**：完全一致（21.3 为 核心规则）
- **日志保留策略**：完全一致（21.5 为 核心规则）

结算列表仅记录其自身入口触发的操作（如人工结算的快捷通道、人员分配）；所有操作统一汇入同一日志库，按"来源页面"字段区分触发入口。
