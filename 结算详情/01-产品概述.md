# 第一章 产品概述

> **关盘口径（2026-04-21 生效）**：关盘来源统一为数据源推送（唯一来源）；关盘 = 绝对终态，操盘页与结算详情页均不提供人工开盘入口。

## 1.1 产品定位

结算详情页是单场赛事的结算操作页面，负责赛果录入、盘口级结算执行、驳回与二次结算、赛事异常处理和赛事级确认结算。

| 定位项 | 说明 |
|--------|------|
| 所属系统 | P1 体育操盘系统 |
| 适用赛事类型 | **仅常规单场赛事**（EventTypeId = 2）|
| 核心目标 | 完成单场赛事的全部盘口结算，确保赛果准确、派彩正确 |
| 上游 | 结算列表 · 常规赛事 Tab（点击赛事行进入）|
| 下游 | 财务系统（派彩发放 / 差额回收）|
| 数据源口径 | 仅 IM（Inplay Matrix）为唯一数据源 |
| **姐妹模块** | 冠军赛事 / 长期盘口（EventTypeId = 1）见 [冠军赛事结算详情](/settlement-outright/) |
| 盘口级结算前置 | 盘口已关盘 + 该盘口所需赛果已确定（可以来自数据源同步或人工录入）|
| 赛事级结算前置 | 该赛事下所有盘口均已结算或已取消 |

> **关于数据源对比赛结束的输出**：
> - 数据源**不单独提供**独立的"赛事状态枚举字段"（类似"比赛已结束"这种赛事级状态标志）
> - 数据源**确实提供"比赛结束时间"时间戳字段**：在完整赛果数据中返回
> - 数据源还通过增量推送返回"赛事移除"信号，表示赛事从活跃列表中移除
> - 本地结合"比赛结束时间字段"+"赛事从活跃列表移除"共同感知比赛是否结束
>
> 各盘口按自身时段独立结算——半场盘口可在半场结束后结算，15 分钟段盘口可在该段结束后结算，不必等全场终场。赛事级确认结算仅要求所有盘口完成结算。

## 1.2 与结算列表的分工

| 维度 | 结算列表 | 结算详情页 |
|------|---------|-----------|
| 定位 | 多场赛事/多笔注单的浏览与快捷操作入口 | 单场赛事的完整结算工作台 |
| 结算方式 | 注单视图：直接为注单选择结算结果（**快捷通道**，结果应用到对应盘口） | 录入赛果 → 系统计算结果 → 逐盘口确认（**完整操作**） |
| 本质区别 | 是入口/快捷方式，不是独立结算机制 | 是结算的主工作台，支持全部操作 |
| 赛事级操作 | 无（只展示和导航） | 确认结算、赛事驳回、赛事异常处理（4 选 1：取消 / 延期 / 腰斩 / 数据错误）|
| 赛果录入 | 无 | 完整的赛果数据录入弹窗（比分/进球/角球/红黄牌） |
| 适用场景 | 快速为单笔注单选择结算结果（任何 L 级） | 完整处理一场赛事的全部盘口 |
| 优先级 | — | **详情页优先**。若两边同时操作同一盘口，以详情页为准 |
| 驳回 | 不提供驳回入口 | 赛事驳回 + 盘口二次结算入口统一在详情页执行 |

## 1.4 两条结算路径

同一赛事中，不同盘口可能走不同路径：

### 路径 A：数据源结算（IM 自动）

适用条件：操盘时数据源开关 = 开启

```
### 路径 B：本地结算（人工）

适用条件：盘口已关盘，且数据源未推送该盘口的终态（比分/状态字段），或该 BT 为数据源无法覆盖类型

```
数据源或结算人员提供该盘口所需赛果（全场比分、半场比分、某段比分、进球详情等）
→ 结算人员在「赛果录入」弹窗补录 / 修正数据（或系统从数据源自动同步）
→ 系统根据录入数据计算对应盘口的结算结果
→ 结算人员逐盘口确认 → 执行结算

（允许分阶段：半场结束后可录入半场比分并结算半场盘口，不必等全场终场）
```

### 同一赛事两条路径并存

| 场景 | 路径 A 的盘口 | 路径 B 的盘口 |
|------|-------------|-------------|
| IM 全量推送比分及盘口终态 | 全部 | 无 |
| IM 部分推送比分/盘口终态 | IM 已推送终态的盘口 | IM 未推送终态的盘口 |
| IM 无法覆盖的 BT | IM 能覆盖的 | IM 无法覆盖的（40 种） |

## 1.5 核心术语（三级层次）

与[操盘列表](/trading-list/) 核心规则对齐的三级术语层次：

| 术语 | 英文 | 行业术语 | 含义 | 示例 |
|------|------|---------|------|------|
| **玩法** | Bet Type | Bet Type / Market Type | 抽象的投注类型，由 IM BetTypeId 标识 | "全场让球"（BetTypeId = BT1）|
| **盘口** | Market | Market / Line | 具体市场实例，一个玩法可对应多个盘口（按盘口线区分）| "BT1 全场让球 -0.5"、"BT1 全场让球 -1.0" |
| **选项** | Selection | Selection / Outcome | 具体可下注的选择 | "主队 -0.5"、"客队 +0.5" |

**约定**：本文档及 settlement 模块全文遵循此三级术语。
- **BT（Bet Type）**：玩法类型的行业缩写，由 IM 数据源的 `BetTypeId` 字段定义（如 BT1 = 全场让球、BT6 = 全场波胆）
- 当语境指代"某个玩法下的具体盘口线"时，用"盘口"；指代玩法类型时，用"玩法"
