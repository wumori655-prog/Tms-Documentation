# 第十四章 串关结算规则

> **关盘口径（2026-04-21 生效）**：关盘来源统一为数据源推送（唯一来源）；关盘 = 绝对终态，操盘页与结算详情页均不提供人工开盘入口。

## 14.1 串关术语

| 术语 | 含义 |
|------|------|
| 母单 | 包含多个投注选项的串关注单 |
| 子单 | 母单中每一个单独的投注选项 |
| M串N | M 个子单中取 N 个组合。M串1 = 全部 M 个子单组成 1 注 |
| 子单系数 | 子单结算后用于计算母单派彩的系数 |

---

## 14.2 母单结算触发条件

| 规则 | 说明 |
|------|------|
| 常规触发 | 母单所有子单均已完成结算 → 系统自动计算母单派彩，**无需结算人员确认** |
| 提前结算（短路） | 任一子单结算为「输」→ 母单派彩必为 0，系统立即结算母单为「输」，无需等待其余子单 |
| 分段子单短路 | 半场/15分钟等分段玩法在**终场统一结算**时，如子单判定为「输」，立即触发母单短路结算 |
| 部分派彩 | 不支持。未触发短路时，任一子单未结算，母单保持待结算 |

---

## 14.2b 串关盘口结算规则（按子单盘口来源分支）

**核心原则**：串关子单的结算结果**沿用其所在盘口的结算结果**（不重复计算）。不同来源的盘口走不同分支：

| 子单所在盘口来源 | IM 推送状态 | 子单结算方式 |
|----------------|------------|-------------|
| **可操盘盘口** | —（不适用）| 本地引擎按 [第 12 章](./12-BT级结算判定规则) 判定规则基于比分结算 |
| **数据源盘口** | IM 超时未推送盘口终态 + 该盘口进入本地锁定态 | 本地引擎按第 12 章规则基于比分兜底计算；结算人员通过结算详情页完成 |

**术语统一**：全章节使用"母单 / 子单"指代串关结构，**不使用"腿"**。

**规则说明**：

| 规则 | 说明 |
|------|------|
| 子单结算不重复计算 | 子单结算 = 盘口结算结果派生；盘口结算完成即子单结算完成 |
| 母单派彩公式 | 投注额 × 各子单系数连乘（见 第 14.3 节 子单系数定义、第 14.4 节 母单派彩公式）|
| 数据源盘口进入本地锁定后 | 该盘口所有单关 + 串关子单一律由本地引擎处理，IM 后续推送的比分/状态变化不再覆盖本地结果（见 [第 8.3 节 本地锁定态](./08-数据源盘口#_8-3-数据源接管态-与-本地锁定态-单向流转)）|
| 比分数据来源一致性 | 无论子单走哪个引擎，原始数据输入（比分、进球、角球等）均为**同一份 IM 推送数据**（数据源正常时）或 **同一份结算人员录入数据**（数据源超时时）；保证混合串关子单的数据视角一致 |
| 母单结算时序 | 等所有子单均有结算结果（无论来源）后触发母单派彩计算；任一子单判输立即触发母单短路（见 第 14.2 节）|

---

## 14.2c 子单模型（核心概念）

| 概念 | 定义 |
|------|------|
| 子单 | 母单中一条投注选项的引用记录，本质是"对某个盘口的某个选项下注" |
| 子单赔率 | 投注瞬间从数据源快照锁定，后续不随数据源赔率变动 |
| 子单结算结果 | 由子单引用的盘口对该选项的结算结果派生（该盘口判赢则子单系数 = 投注瞬间锁定 Decimal，判输则系数 = 0，以此类推） |
| 子单状态 | 没有独立状态，完全跟随引用盘口该选项的结算状态 |

**关键规则**：所有子单结算必定由**引用盘口的赛果**派生。盘口赛果未变 → 子单系数不变 → 母单派彩不变。盘口赛果发生改变 → 子单系数重算 → 母单派彩重算 → 按 15.4 差额矩阵处理派彩差额。

**同赛事不串关**：为避免对冲风险，**同一赛事**（指同一 EventId，不论 EventTypeId = 1 或 2）不同盘口不可出现在同一母单中。不同 EventId 的盘口（例如两场常规赛事，或一场常规 + 一场冠军）即使属于同一联赛也可同母单（详见 14.2d 跨类型串关）。此约束在投注层校验，结算详情页不做重复校验。

---

## 14.2d 跨类型串关（常规赛事 + 冠军赛事子单）

母单可能同时包含常规赛事子单（EventTypeId=2）和冠军赛事子单（EventTypeId=1）。两类赛事结算节奏差异巨大(常规赛事几小时内结算,冠军赛事可能跨数月),规则如下:

| 规则 | 说明 |
|------|------|
| 统一本地结算引擎 | 两类子单均由本地结算引擎处理，引擎规则一致 |
| 母单等最晚子单 | 母单状态保持"待结算"直到**所有子单**（含冠军子单）结算完成；派彩可能延迟数月 |
| 单个子单结算不影响母单 | 任一子单结算（含冠军）后，母单按规则重新评估：若还有未结算子单 → 保持待结算；若全部结算完 → 触发派彩 |
| 冠军子单被二次结算时 | 同常规子单规则：引用盘口赛果变化 → 母单回退为"待结算·子单回退中" → 新赛果确定后重算派彩 |
| 冠军盘口取消时 | 该子单系数 = 1.0（降级），母单按 14.5 降级规则处理 |
| 锦标赛整体取消时 | 该场所有子单系数 = 1.0（降级），串关退款 |

### 跨类型串关示例

**母单**：投注 ¥100，3 串 1：
- 子单 1：本周末某场英超比赛让球盘（常规，EventTypeId=2，HK 0.88）
- 子单 2：英超冠军「曼城」选项（冠军，EventTypeId=1，HK 1.85）
- 子单 3：本周末另一场西甲比赛独赢盘（常规，HK 0.95）

**结算过程**：
```
周末 → 常规赛事比赛结束 → 子单 1 和子单 3 结算（比如都赢）
  → 母单仍是"待结算"（子单 2 未结算）
  → 母单保持待定状态，等数月后冠军盘口结算

次年 5 月赛季末 → 曼城提前锁定冠军 → 结算人员结算英超冠军盘口
  → 子单 2 结算（赢）
  → 母单按规则触发派彩：¥100 × 1.88 × 2.85 × 1.95 = ¥1,046.00
  → 向下取整 2 位 = ¥1,046.00
```

### 关联日志

跨类型串关挂起期间，系统在结算日志中记录「母单 {ID} 待冠军子单 {盘口 ID} 结算」；每次有子单结算都更新日志条目。

## 14.3 子单系数定义

| 子单结算结果 | 子单系数 | 对母单的影响 |
|------------|---------|------------|
| 赢 | Decimal（HK + 1） | 正常计入 |
| 输 | 0 | 母单派彩 = 0（整单输） |
| 赢半 | 0.5 × HK + 1 = 0.5 × Decimal + 0.5 | 按赢半系数计入 |
| 输半 | 0.5 | 按输半系数计入 |
| 走水 | 1.0 | 等同于该子单移除 |
| 取消 | 1.0 | 子单降级（串关降级） |
| 退款 | 1.0 | 子单降级（串关降级） |

---

## 14.4 母单派彩公式

```
母单派彩 = 投注额 × 子单1系数 × 子单2系数 × ... × 子单N系数
```

派彩向下取整至 2 位小数。

**赢半系数等价推导**：

```
单关赢半派彩 = 投注额 + 投注额 × 0.5 × HK = 投注额 × (1 + 0.5 × HK) = 投注额 × (0.5 × HK + 1)
所以子单系数 = 派彩 ÷ 投注额 = 0.5 × HK + 1

等价变换：
  0.5 × HK + 1
= 0.5 × (Decimal - 1) + 1
= 0.5 × Decimal - 0.5 + 1
= 0.5 × Decimal + 0.5

示例：HK = 0.92，Decimal = 1.92
  系数 = 0.5 × 0.92 + 1 = 1.46
  验证 = 0.5 × 1.92 + 0.5 = 1.46 ✓
```

## 14.4b 本金不变原则

### 14.4b.1 核心规则

不论子单结果如何变化（赢 / 输 / 取消 / 走盘 / 退款），母单本金（投注额）**永远等于初始投注金额**，不发生改变。

### 14.4b.2 哪些会变 / 哪些不变

| 项目 | 是否会变 | 说明 |
|---|---|---|
| 本金（投注额）| **不变** ❌ | 投注时锁定，不论任何事件 |
| 各子单系数 | 会变 ✅ | 按 14.3 子单系数定义计算 |
| 派彩公式 | 会变 ✅ | 系数变化导致公式结果变化（详见 14.4）|
| 最终派彩 | 会变 ✅ | 公式计算结果 |

### 14.4b.3 数值示例

**情境**：玩家投注 ¥30，3 场串关（A × B × C），赔率分别 1.88 / 1.92 / 2.50

| 事件 | 本金 | 派彩计算 | 派彩 |
|---|---|---|---|
| 投注 | ¥30 | 30 × 1.88 × 1.92 × 2.50 = 270.72 | 待定 |
| 3 场全赢 | **¥30** | 30 × 1.88 × 1.92 × 2.50 | ¥270.72 |
| B 场取消 | **¥30** | 30 × 1.88 × 1.0 × 2.50 | ¥141.00（降级为 2 串 1）|
| B 场输 | **¥30** | 30 × 1.88 × 0 × 2.50 | ¥0（玩家输本金）|
| 全部取消 | **¥30** | 30 × 1 × 1 × 1 | ¥30（全额退款）|
| 全部走水 | **¥30** | 30 × 1 × 1 × 1 | ¥30（全额退款）|

→ 注意：「本金」栏在所有情境下永远 ¥30。

### 14.4b.4 对统计的意义

由于本金不变，下列统计场景**可以放心使用本金（投注额）作为基准**：

| 统计场景 | 用本金 | 理由 |
|---|---|---|
| 下架弹窗「当前投注额」 | ✅ | 本金恒定，数字稳定 |
| 锁盘弹窗「将影响投注额」 | ✅ | 本金恒定 |
| 顶部栏「总投注额」 | ✅ | 本金恒定 |
| 风控敞口最大值 | ✅ | 用本金作为下限风险 |

但若统计需要「最大可能派彩」（公司最大潜在赔付），则需按当下各子单系数动态计算，**不能使用初始公式**。

## 14.5 串关降级规则

| 原始串关 | 取消/退款子单数 K | 有效子单数 | 处理 |
|---------|-----------------|----------|------|
| N 串 1 | K < N-1 | N-K ≥ 2 | 降级为 (N-K) 串 1 |
| N 串 1 | K = N-1 | 1 | 降级为单关 |
| N 串 1 | K = N | 0 | 所有子单系数 = 1.0，母单按派彩公式计算后等于投注额，系统视同已取消并全额退还投注额 |

> **母单无独立取消操作**：母单状态不可被人工直接取消；母单"已取消"态由其所有引用盘口全部取消（或走水 / 退款降级）时，按派彩公式计算得出，系统自动完成退款。

---

## 14.6 数值示例

### 3串1，全赢

投注额 ¥1,000：

| 子单 | HK | Decimal | 结果 | 系数 |
|------|-----|---------|------|------|
| 子单1 | 0.88 | 1.88 | 赢 | 1.88 |
| 子单2 | 2.10 | 3.10 | 赢 | 3.10 |
| 子单3 | 0.95 | 1.95 | 赢 | 1.95 |

派彩 = 1,000 × 1.88 × 3.10 × 1.95 = 1,000 × 11.3646 = ¥11,364.60
验证：1.88 × 3.10 = 5.828 × 1.95 = 11.3646 ✓

### 3串1，1子单取消

投注额 ¥1,000：

| 子单 | 结果 | 系数 |
|------|------|------|
| 子单1 | 赢 | 1.88 |
| 子单2 | 取消 | 1.00 |
| 子单3 | 赢 | 1.95 |

派彩 = 1,000 × 1.88 × 1.00 × 1.95 = ¥3,666.00（等同于 2串1）

### 3串1，含输

任一子单输 → 系数 = 0 → 母单派彩 = 0。

## 14.7 引用盘口赛果变化时的串关处理

当母单某子单引用的盘口发生赛果改变时（盘口二次结算并重选了不同结果；或 IM 推送了新比分导致本地重新判定结果变化）：

| 步骤 | 说明 |
|------|------|
| 1 | 该盘口对该子单选项的结算结果变更为新值 |
| 2 | 包含该子单的所有母单回退为待结算·等子单 |
| 3 | 其他子单保持原态（其引用盘口赛果未变） |
| 4 | 该盘口新结算结果确定后，母单按新子单系数重新计算派彩 |
| 5 | 按 15.4.1 差额公式处理母单派彩差额：差额 > 0 补发；差额 < 0 扣回（不足记坏账）；差额 = 0 不处理（子单系数无变化时跳过）|

> **盘口状态变化但赛果未变（如数据源推送开盘自动回滚、二次结算后重选相同结果）不触发母单级联**。盘口赛果保持原值时，子单系数不变，母单派彩不变，无需任何操作。

## 14.8 复式串关

复式串关 = 同一组选项按多种组合投注，每个子注独立结算。

| 规则 | 说明 |
|------|------|
| 子注独立 | 每个子注独立计算派彩 |
| 子注投注额 | 每注等额 |
| 总派彩 | 所有子注派彩之和 |
| 精度 | 每注独立向下取整后求和（不可先加后取整） |

### 14.8.1 复式串关降级规则

当复式串关中某选项取消/退款时，包含该选项的子注按 N串1 降级规则处理：

| 场景 | 处理 |
|------|------|
| 3选项复式（含2串1×3 + 3串1×1），1选项取消 | 含该选项的2串1子注→降级为单关（系数=1.0）；3串1子注→降级为2串1 |
| 3选项复式，2选项取消 | 含2个取消选项的3串1子注→降级为单关；含1个取消选项的2串1子注→降级为单关；2个取消选项互含的2串1子注→全退 |

**数值示例**：3选项复式，每注 ¥100，选项A/B/C（HK 0.88/2.10/0.95），选项B取消

| 子注 | 组合 | 选项B系数 | 降级结果 | 派彩 |
|------|------|----------|---------|------|
| 2串1-AB | A+B | 1.00 | 降为单关A | 100 × 1.88 = ¥188.00 |
| 2串1-AC | A+C | — | 不受影响 | 100 × 1.88 × 1.95 = ¥366.60 |
| 2串1-BC | B+C | 1.00 | 降为单关C | 100 × 1.95 = ¥195.00 |
| 3串1-ABC | A+B+C | 1.00 | 降为2串1 AC | 100 × 1.88 × 1.95 = ¥366.60 |
| **总派彩** | | | | **¥1,116.20** |
