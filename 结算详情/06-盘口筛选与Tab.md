# 第六章 盘口筛选与 Tab

## 6.1 Tab 结构

| Tab | 标识 | 数量标签 | 内容 |
|-----|------|---------|------|
| 可操盘盘口 | localTab | 17 种 | 全部 17 种 BT 可操盘盘口结算卡片（UI 简写「本地操盘」）|
| 数据源盘口 | datasourceTab | N 种 | 结算完全依赖 IM 驱动的盘口。默认处于数据源接管态（只读，本地按 IM 状态字段派彩）；该盘口已关盘 + 所需比分数据已可得 + 数据源超时未推送盘口终态后，解锁为「待手动结算」，本地结算后进入本地锁定态（单向不可逆，不再接受 IM 推送覆盖） |

## 6.2 盘口筛选工具栏

| 筛选维度 | 选项 | 默认值 | 说明 |
|---------|------|--------|------|
| 赛事阶段 | 全部 / 滚球 / 赛前 | 全部 | 滚球按钮带红色脉冲动画 |
| 赛制时段 | 全部 / 上半场 (1H) / 下半场 (2H) / 15 分钟段 / 全场 (FT) / 事件触发 | 全部 | 6 tab，覆盖时段维度 3 类（1H / 2H / FT）+ 15 分钟段 + 事件触发维度；英文缩写对齐 IM 字段 |
| 结算状态 | 全部 / 开盘中 / 待结算 / 已结算 / 已取消 | 全部 | 盘口无"异常"状态（异常为赛事级，盘口通过"已结算"或"已取消"体现异常处理结果）|
| 盘口搜索 | 文本输入 | 空 | 支持按 BT 编号搜索（如 BT1、BT6） |

> **赛制时段 tab 的卡片归类规则**：
> - **上半场 (1H)**：1H 比分即可结算的盘口（BT1-1H / BT2-1H / BT6-1H / BT9 半全场上半部分）
> - **下半场 (2H)**：依赖下半场比分的盘口（BT1-2H / BT2-2H / BT6-2H / BT24 / BT25 / BT26 / BT27 / BT28 / BT29 / BT30 / BT33 / BT34 / BT57 / BT19 / BT20 / BT291 等）。**下半场比分由下半场赛果录入弹窗独立录入**（上半场已应用后启用；数据源开关 = 开时预填 = 数据源全场 − 半场，可核对 / 修正）
> - **15 分钟段**：6 段独立判定的盘口（BT48 / BT179 / BT180）
> - **全场 (FT)**：依赖 FT 比分的盘口（BT1 / BT2 / BT3 / BT5 / BT7 / BT8 / BT158 / BT160 / BT161）
> - **事件触发**：依赖进球明细或实时比分的盘口（BT37 第 X 粒进球队伍 / BT159 角球第 X 粒 / BT6 赛中判输）
>
> 赛果录入按钮（筛选行右侧）按当前时段 tab 动态显示：
> - 上半场 → 上半场赛果录入
> - 下半场 → 下半场赛果录入（前置：上半场已应用）
> - 15 分钟段 → 15 分钟段赛果录入
> - 全场 → 全场赛果与扩展（前置：上半场 + 下半场均已应用）
> - 事件触发 → 无独立按钮（进球明细在上半场 / 下半场 / 全场扩展弹窗按时段分别录入）

筛选结果展示计数：如「筛选结果: 8 / 17 个盘口」。
