## 17.2 权限设计要点

| 要点 | 说明 |
|------|------|
| 结算人员 | 一线结算操作：赛果录入 / 盘口结算 / 赛事驳回 / 盘口二次结算 / 赛事异常处理 |
| 结算主管 | 一线操作 + 管理级操作：结算人员分配 / 批量分配（入口在结算列表）|
| 与操盘手的关系 | **关盘来源为数据源推送**（唯一来源）；操盘页与结算页面均不提供关盘入口；关盘 = 绝对终态，无人工开盘入口。职责分离：操盘手负责赔率与隐藏/锁定，结算人员负责赛果/结算/二次结算/异常处理 |
| 数据源盘口关盘/开盘 | 架构约束，任何角色均不可人工触发；数据源盘口的生命周期完全由数据源自身驱动 |
| 赛事驳回时效 | 所有赛事驳回操作须在赛事最新结算时间起 24 小时内发起（**按 UTC 时间戳**），超出窗口按钮禁用 |
| 赛事异常发起 | 仅赛事状态 = 待结算时可发起（取消 / 延期 / 腰斩 / 数据错误，4 选 1）。系统不会自动标记 |
| 风控 / 财务 | 仅查看权限，不参与结算操作（一期无导出功能）|

## 17.3 结算人员分配

结算人员的指派**按阶段分配**（赛前结算人员 + 滚球结算人员，每场赛事分别指定），由结算主管在结算列表中维护。本章不展开；完整弹窗结构与交互见[结算列表第 8 章 8.3 节 结算人员分配弹窗](/settlement/08-弹窗交互#_8-3-结算人员分配弹窗)。

> **与操盘手分配的区分**：操盘手分配**按玩法**（17 种 BT 分别指定），在操盘列表维护；结算人员分配**按阶段**，在结算列表维护。两者数据模型不同，不可混淆。
