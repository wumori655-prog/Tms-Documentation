## 结算判定规则速查（按BT类型）

以下规则适用于所有时段（全场/上半场/下半场/15分钟段），区别仅在于"基准分"的取值不同。

### 基准分定义

| 时段 | 赛前注单基准分 | 滚球注单基准分 |
|------|-------------|-------------|
| 全场(FT) | 终场比分 X:Y | 下注后比分 = 终场比分 − 下注时比分 |
| 上半场(1H) | 半场比分 X:Y | 下注后比分 = 半场比分 − 下注时比分 |
| 下半场(2H) | 下半场净比分 = 全场 − 半场 | 下注后比分 = 下半场净比分 − 下注时净比分 |
| 15分钟段 | 该段比分 X:Y | 下注后比分 = 该段比分 − 下注时比分 |
| 角球 | 同上，主/客替换为角球数 | 同上 |

> **滚球受影响的BT**：BT1(让球)、BT2(大小)、BT32(客队大小)、BT160(主队大小)、BT161(角球客队大小)、BT179(15分钟让球)、BT180(15分钟大小)
> **不受影响的BT**（赛前和滚球均用该时段最终比分）：BT3(独赢)、BT5(单双)、BT6(波胆)、BT7(总进球)、BT8(双重机会)、BT9(半全场)、BT158(反波胆)、BT159(第X粒进球队伍)

### 多基准分场景说明

- 每笔注单的"下注时比分"在投注瞬间由数据源随注单返回的当前比分字段记入注单表
- **同盘口内注单的"下注时比分"可能不同**（当数据源在比分变化后对该盘口走"暂停-恢复"、而非"关旧挂新"时）
- 结算时系统按每笔注单独立计算基准分；下注时比分相同的注单自动归为一组
- **结算 UX 按盘口内基准分组数量自适应**：
  - 1 组（约 90% 场景）→ 走单组 UX，弹窗一次选定全盘口结果（与现有 UX 一致）
  - ≥ 2 组（约 10% 场景）→ 走多组 UX，弹窗按组分区，每组独立选定、同组内联动、跨组独立
- 完整自适应规则见[结算详情页第 7.3.6 节 自适应基准分组结算 核心规则](/settlement-detail/07-盘口结算卡片#_7-3-6-自适应基准分组结算规则-核心规则)；典型场景示例见[结算场景手册 N7](/settlement-detail/附录-结算场景手册#n7-滚球让球多组基准分场景-单关)

### R1. 让球类（BT1、BT179 及所有角球让球）

**输入**：主队基准分 H、客队基准分 A、盘口线值 L

**公式**：净差值 = H − A + L

| 盘口线类型 | 净差值条件 | 主队选项 | 客队选项 |
|-----------|----------|---------|---------|
| 半球线(x.5) | > 0 | 赢 | 输 |
| 半球线(x.5) | < 0 | 输 | 赢 |
| 四分之一球(x.25/x.75) | > 0.25 | 赢 | 输 |
| 四分之一球 | = 0.25 | 赢半 | 输半 |
| 四分之一球 | = −0.25 | 输半 | 赢半 |
| 四分之一球 | < −0.25 | 输 | 赢 |
| 整数线(x.0) | > 0 | 赢 | 输 |
| 整数线(x.0) | = 0 | 走水 | 走水 |
| 整数线(x.0) | < 0 | 输 | 赢 |

**示例（半球线）**：终场 2:1，盘口线 −0.5 → 净差值 = 2−1+(−0.5) = 0.5 > 0 → 主赢客输
**示例（四分之一球）**：终场 2:1，盘口线 −0.75 → 净差值 = 2−1+(−0.75) = 0.25 → 主赢半，客输半
**示例（整数线）**：终场 2:1，盘口线 −1.0 → 净差值 = 2−1+(−1.0) = 0 → 走水
**滚球示例**：1:0时下注，终场 2:1 → 下注后比分 (2−1):(1−0) = 1:1 → 净差值 = 1−1+(−0.5) = −0.5 < 0 → 主输客赢

### R1b. 三方让球（BT35 全场/上半场/下半场三方让球，角球三方让球）

**输入**：主队基准分 H、客队基准分 A、让球值 L

**判定**：将让球值加到主队得分后，按独赢1X2逻辑判定。

| 条件（H+L vs A） | 主胜(1) | 平局(X) | 客胜(2) |
|-----------------|---------|---------|---------|
| H + L > A | 赢 | 输 | 输 |
| H + L = A | 输 | 赢 | 输 |
| H + L < A | 输 | 输 | 赢 |

**示例**：终场 1:1，三方让球 主队−1 → 1+(−1)=0 vs 1 → 0<1 → 客胜赢

---

### R2. 大小类（BT2、BT160、BT161、BT32、BT180 及角球大小）

**输入**：基准分、盘口线值 L

**公式**：
- BT2/BT180/角球大小：总分 = 主队基准分 + 客队基准分
- BT160/角球主队大小：总分 = 主队基准分
- BT161/BT32/角球客队大小：总分 = 客队基准分

净差值 = 总分 − L，判定规则与让球类一致（>0 大赢，<0 小赢，=0 走水，含半球/四分之一球规则）

**示例**：终场 2:1，BT2 盘口线 2.5 → 总分 = 3 → 净差值 = 3−2.5 = 0.5 > 0 → 大赢小输
**示例**：终场 2:1，BT160 盘口线 1.5 → 总分 = 2(主队) → 净差值 = 2−1.5 = 0.5 > 0 → 大赢小输

### R3. 独赢1X2（BT3、BT48 15分钟独赢）

**输入**：主队基准分 H、客队基准分 A

| 比分关系 | 主胜(1) | 平局(X) | 客胜(2) |
|---------|---------|---------|---------|
| H > A | 赢 | 输 | 输 |
| H = A | 输 | 赢 | 输 |
| H < A | 输 | 输 | 赢 |

### R4. 单双（BT5 及角球单双）

**输入**：主队基准分 H、客队基准分 A

| 条件 | 单(Odd) | 双(Even) |
|------|---------|---------|
| (H+A) 为奇数 | 赢 | 输 |
| (H+A) 为偶数（含0） | 输 | 赢 |

### R5. 波胆（BT6，含角球波胆）

**输入**：基准分 H:A

**判定**：选项比分 = H:A → 该选项赢，所有其他选项输。有且仅有1个选项赢。

**赛中自动判输**：
1. 比分变化时系统自动扫描所有波胆选项
2. 若当前主队进球 > 选项中主队进球，或当前客队进球 > 选项中客队进球 → 该选项已无法实现 → 标记为输（派彩=0，注单状态=已结算）
3. 若当前比分 = 选项比分但比赛未结束 → **不判定**（等基准分最终确定）
4. **只判输不判赢**："赢"必须等基准分最终确定后才判定
5. "其他比分"选项赛中不判定

**二次结算规则**：BT6 盘口二次结算时**所有注单统一回退**（含赛中已判输的注单）。因为比分若回退（如 VAR 取消进球），原先"已无法中奖"的选项可能重新有效，为保持判定一致性，二次结算**不区分赛中判输 / 终场判定**，一律回退后按新赛果重算

**示例**：终场 2:1 → "2:1"选项赢，"1:0""0:0""1:1""2:0""其他"等全部输

---

### R6. 反波胆（BT158）

**输入**：基准分 H:A
**选项结构**：5×5 + "其他" = 26 个（与 BT6 同结构，对应 IM BT158 BetTypeId 标准数据结构）

**判定**：与波胆取反。选项比分 ≠ H:A → 赢（25 个）；选项比分 = H:A → 输（唯一 1 个）。一期不做赛中结算（等基准分确定后统一判定）。

**示例**：终场 2:1 → 选项"0:0"赢、"1:0"赢、"1:1"赢……选项"2:1"输（唯一匹配的选项输，其余 25 个全赢，"其他"选项按"2:1 未落入矩阵"时赢）

---

### R7. 总进球（BT7，含角球总进球）

**输入**：主队基准分 H、客队基准分 A

**判定**：总进球数 = H + A → 匹配的选项赢，其余输。
- 选项"7+"：当 H+A ≥ 7 时赢

**示例**：终场 2:1 → 总进球=3 → "3球"赢，"0球""1球""2球""4球""5球""6球""7+"全输

---

### R8. 双重机会（BT8，含角球双重机会）

**输入**：主队基准分 H、客队基准分 A

| 比赛结果 | 1X(主或平) | 12(主或客) | X2(平或客) |
|---------|-----------|-----------|-----------|
| H > A(主胜) | 赢 | 赢 | 输 |
| H = A(平局) | 赢 | 输 | 赢 |
| H < A(客胜) | 输 | 赢 | 赢 |

每种结果恰好2赢1输。

---

### R9. 半全场（BT9，含角球半全场）

**输入**：半场结果（主胜/平/客胜）+ 全场结果（主胜/平/客胜）

9种选项组合（主/主、主/平、主/客、平/主、平/平、平/客、客/主、客/平、客/客），实际半场+全场结果匹配的组合赢，其余8个输。

**示例**：半场 1:0(主胜)，终场 2:1(主胜) → "主/主"赢，其余全输

---

### R10. 第X粒进球队伍（BT37、BT159）

**输入**：进球明细列表（每个进球的时间+队伍）

**判定**：
- 第X个进球由主队打入 → "主队"赢
- 第X个进球由客队打入 → "客队"赢
- 总进球数 < X → "无进球"赢

**示例**：第1粒进球为主队第23分钟打入 → BT37"第1球-主队"赢

## 结算自动化层级与本表的关系

| 层级 | 盘口类型 | 赛果来源 | 本表对应 |
|------|---------|---------|---------|
| L1 系统直接执行 | 数据源盘口（单关） | IM 推送盘口终态（比分+状态字段）| 表1~9中不可操盘的玩法 |
| L2 本地按 BT 规则自动计算+结算人员确认 | 可操盘盘口 | IM 推送比分数据 | 表1~9中可操盘的玩法（★标记） |
| L3 人工录入比分 + 本地按规则自动计算 | 可操盘盘口（IM 未推送比分）| 结算人员录入比分 → 本地按 BT 规则自动计算 | 表 1~9 中可操盘的玩法（IM 比分缺失时） |
| L4 逐盘口手动选择 | IM无法覆盖BT（40种） | 无法结构化录入赛果 | 表10（40种） |

**补充说明**：
- **不可操盘 ≠ 无法本地结算**。不可操盘仅表示本地不操盘（不手动调赔率），当 IM 赛果未送达时，系统可对任何 BT 执行本地结算兜底。
- **串关子单按盘口来源分支**（对齐 [第 14.2b 节 本地/数据源/混合三分支](./14-串关结算规则#_14-2b-串关盘口结算规则-按子单盘口来源分支)）：
  - **可操盘盘口子单**：本地引擎按第 12 章判定规则结算
  - **数据源盘口子单 + IM 已推送盘口终态**：走 L1 自动结算（与单关一致，人工不介入）
  - **数据源盘口子单 + IM 超时进入本地锁定态**：本地引擎兜底结算
- **数据源盘口进入本地锁定态后**（超时 + 人工结算），其所有单关 + 串关均由本地引擎统一处理；IM 后续推送的比分/状态变化不再覆盖本地结果。
