# 第四章 冠军盘口卡片

> **冠军赛事模块适用边界（2026-04-21 生效）**：本模块为冠军赛事 / 长期盘口（EventTypeId = 1）专用规则。常规赛事的"关盘 = 绝对终态、无人工开盘入口"规则**不适用于本模块**；冠军盘口的生命周期由运营主导，**支持结算人员人工重新开盘**（详见 [第 6.7 节 重新开盘](./06-二次结算与驳回#_6-7-重新开盘-冠军模块独立规则)）。

## 4.2 卡片 Header

| 元素 | 说明 |
|------|------|
| 盘口名称 | 如「英超冠军」「欧冠夺冠」「英超金靴」|
| 锦标赛标签 | 如「2025/26 赛季」 |
| 盘口来源标签 | 统一显示「本地」（冠军盘一律本地结算）|
| 注单数 | 该盘口下注单总数 |
| 投注额 | 该盘口下注单总投注额 |
| 结算状态标签 | 开盘中（蓝）/ 已关盘（橙）/ 已结算（绿）/ 已取消（灰）|
| 取消按钮 | 盘口级取消 |

## 4.3 卡片 Body

冠军盘口通常是**选项列表形式**（与常规赛事 SingleLineTable 类似但没有"单一玩法卡片级结算按钮"以外的选项级交互）：

| 列 | 说明 |
|----|------|
| 选项 | 如「曼城」「阿森纳」「利物浦」等候选队伍 / 球员 |
| 赔率 | 该选项的 HK 赔率 |
| 注单 | 该选项下注单数 |
| 投注额 | 该选项下注单总投注额 |
| 结算结果 | 下拉框（**赢 / 输 / 退款 / 取消**）+ 首选项为「请选择」|
| 派彩额 | 该选项的累计派彩，结算后填充 |

### 4.3.1 结果联动规则（多选项玩法）

冠军盘口通常是"单一赢家"类玩法：

| 规则 | 说明 |
|------|------|
| 单一赢家 | 仅**一个**选项可被标记为「赢」；选择某选项为「赢」后，其他选项自动填入「输」（结算人员可修改）|
| 全部退款 | 若冠军赛事整体取消 → 所有选项自动填入「退款」（不可修改）|
| 部分选项取消 | 例外情况（例如某候选球员赛季前退役）：可将该选项单独标为「取消」，其他选项照常判定 |

### 4.3.2 结算结果下拉选项

| 结果 | 派彩公式 | 业务含义 |
|------|---------|---------|
| 赢 | 投注额 × Decimal 赔率 | 选项命中 |
| 输 | 0 | 选项未命中 |
| 退款 | 投注额 | 全额退还本金（例如盘口取消）|
| 取消 | 投注额 | 全额退还本金（例如选项失效，候选球员退役）|

> 冠军盘口**不支持**赢半 / 输半 / 走水（这些只适用于让球 / 大小类盘口，冠军盘口是单一赢家类）。

## 4.4 卡片 Footer

| 字段 | 计算口径 | 说明 |
|------|---------|------|
| 投注额 | Σ（该盘口所有注单投注额）| — |
| 预计派彩 / 已派彩 | Σ（该盘口所有注单派彩额）| 未结算显示「—」，已结算显示合计 |
| 平台盈亏 | 投注额 - 派彩额 | 蓝色盈利 / 红色亏损 |

## 4.5 操作按钮

| 盘口状态 | 结算按钮 | 二次结算按钮 | 重新开盘按钮 | 取消按钮 |
|---------|---------|------------|------------|---------|
| 开盘中 | 不显示 | 不显示 | 不显示 | 可见（谨慎操作） |
| 已关盘 + 未选择结果 | 禁用 | 不显示 | 显示「重新开盘」 | 可见 |
| 已关盘 + 已选择结果 | 启用（蓝色）| 不显示 | 显示「重新开盘」 | 可见 |
| 已结算 | 灰色「已结算」不可点击 | 显示「二次结算」（距该盘口结算时间 ≤ 24h 启用，超窗禁用）| 不显示（需先二次结算）| 可见 |
| 已取消 | 不显示 | 不显示 | 不显示 | 不显示 |

**关键约束**：
- 冠军盘口**全部是本地可操盘盘口**，「重新开盘」按钮对所有冠军盘口都适用
- 冠军盘口的**「二次结算」按钮时效按该盘口结算时间独立计算**，不与赛事级挂钩
- 盘口取消按钮在开盘中也可用（例如某候选球员退役需要单独取消该盘口而不是整个赛事）

## 4.6 结算操作反馈

| 操作 | 成功反馈 | 失败反馈 |
|------|---------|---------|
| 选择选项结果 + 点击「结算」 | 盘口状态变为已结算（绿色），按钮变灰，派彩数据更新 | 红色提示「结算失败：{原因}」+ 重试按钮 |
| 点击「重新开盘」 | 盘口 已关盘 → 开盘，回到操盘页显示 | 红色提示 + 重试按钮 |
| 点击「二次结算」 | 弹出确认弹窗 → 确认后盘口回退为已关盘 | 红色提示 + 重试按钮 |
| 点击「取消」 | 盘口 → 已取消，注单全额退款；已结算盘口需走扣回派彩流程 | 红色提示 + 重试按钮 |

## 4.7 结算前置条件

| 条件 | 说明 |
|------|------|
| 盘口已关盘 | 关盘来源：数据源 IM 推送投注截止信号（唯一来源，回退历史版本的手动关盘能力）|
| 业务条件已满足 | 结算人员根据官方公告判定（如赛季结束、某队提前夺冠、某球员最终获奖等）|

> 冠军赛事的"业务条件满足"**不依赖数据源赛果推送**（数据源不下发"某队夺冠"这样的结论性事件）。结算人员以官方公告为准，人工判断。

## 4.8 按钮粒度硬约束

| 规则 | 说明 |
|------|------|
| 所有操作按钮均在**玩法级** | 冠军盘口结构简单（单一选项列表），结算按钮只有一个，位于卡片底部 |
| 选项级禁止操作按钮 | 每个候选选项行只有「结算结果」下拉框，不承载任何按钮 |
| 与常规赛事一致的硬约束 | 结算 / 二次结算 / 重新开盘 / 取消等按钮**不得**出现在选项行 |
