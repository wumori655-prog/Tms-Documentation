# 第六章 赛事结算明细表

## 6.1 表格定位

赛事结算明细表是本页的核心区域，一行代表一场已经进入报表范围的赛事。表格既承担结果展示，也承担进一步钻取的入口职责。

---

## 6.2 表头字段

| 列 | 说明 |
|----|------|
| 赛事详情 | 联赛标签 + 对阵 + 比分/异常结果 + 赛事 ID + 结算来源 |
| 结算状态 | 已结算 / 已取消 |
| 盘口数 | 该赛事进入结果统计的盘口数量 |
| 注单数 | 按单关 / 串关两行展示 |
| 投注人数 | 该赛事下去重用户数 |
| 投注额 | 赛事聚合投注额 |
| 派彩额 | 赛事聚合派彩额 |
| 盈亏 | 平台盈亏 |
| 盈利率 | 平台盈亏率 |
| 结算人员 | 最终负责结算的人 |
| 开赛时间 | 赛事开始时间 |
| 结算时间 | 赛事完成结算或取消的时间 |
| 操作 | 跳转注单记录 |

---

## 6.3 赛事详情单元格

赛事详情列由两行组成：

| 层级 | 内容 |
|------|------|
| 主行 | 联赛标签 + 主客队 + 比分/异常结果 |
| 副行 | `赛事ID` + `结算来源标签` |

---

## 6.4 标签展示规则

| 标签 | 展示规则 |
|------|---------|
| 联赛标签 | 用颜色区分联赛，如英超 / 西甲 / 意甲 / 德甲 |
| 结算来源标签 | 数据源结算使用默认来源标签；本地结算使用本地标签 |
| 结算状态标签 | 已结算为绿色，已取消为橙色 |
| 比分异常态 | 腰斩等异常结果不展示数字比分，直接展示文本结果 |

---

## 6.5 注单数单元格

注单数列分为两行：

| 子字段 | 说明 |
|--------|------|
| 单关 | 该赛事关联的单关注单数 |
| 串关 | 该赛事关联的串关注单数 |

> 此处用于展示订单结构分布，便于识别“单关驱动”还是“串关驱动”的赛事。

---

## 6.6 金额列规则

| 列 | 规则 |
|----|------|
| 投注额 | 显示为人民币格式，千分位分隔 |
| 派彩额 | 与投注额同格式，视觉上单独高亮 |
| 盈亏 | `>0` 绿色，`<0` 红色，`=0` 灰色 |
| 盈利率 | 以百分比展示，跟随盈亏颜色 |

---

## 6.7 汇总行

表格底部提供当前页汇总行：

| 汇总字段 | 规则 |
|---------|------|
| 赛事数 | 显示“合计（N场赛事）” |
| 盘口数 | 当前页盘口数合计 |
| 注单数 | 当前页单关/串关合计 |
| 投注人数 | 当前页用户数汇总值 |
| 金额字段 | 当前页金额合计 |
| 非可汇总字段 | 状态 / 结算人员 / 开赛时间 / 结算时间 / 操作 列显示 `-` |

---

## 6.8 行操作

| 操作 | 说明 |
|------|------|
| 注单记录 | 跳转到注单记录页，并带入 `matchId` 作为查询条件 |

当前原型只定义跳转行为，不在本模块内继续定义注单记录页内容。
